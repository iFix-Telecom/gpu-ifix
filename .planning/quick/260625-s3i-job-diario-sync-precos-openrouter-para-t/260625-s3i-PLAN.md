<objective>
Build a versioned, idempotent, fail-safe daily job that syncs OpenRouter reference pricing and the live USD/BRL forex rate into the gateway's `ai_gateway.prices` / `ai_gateway.fx_rates` tables via `gatewayctl`. This makes the dashboard's `cost_local_phantom_brl` non-zero (today R$0 because the only seeded price row is keyed `qwen3.5-27b`, which matches none of the actual billed model strings — the dominant local traffic is keyed `model.gguf`).

Purpose: the gateway's cost-attribution lookup is an EXACT `{model, provider, unit}` map-key match. The local pod reports `model.gguf` verbatim, so a phantom reference price row keyed `(model=model.gguf, provider=openrouter-fireworks, unit=input_token|output_token)` is what turns GPU-saved-cost reporting on. Pricing must be fetched live (OpenRouter slugs move), forex must be fetched live (seed is stale at 5.10), and a fetch failure must never clobber a good existing row.

Output: a script in the repo (`scripts/price-sync/`), a systemd user `.service` + `.timer` mirroring `prod-primary-report.*`, and an operation/deploy README. The orchestrator installs the timer on ops-claude and smoke-tests after this plan lands; the executor only creates + commits the files (does NOT install the unit on the live host).
</objective>

  <action>
Create a bash script with `#!/usr/bin/env bash` and `set -euo pipefail`. Header JSDoc-style comment block: what it does, where it runs (ops-claude), how it reaches the gateway, and that pricing/forex are fetched live.

Structure and required behavior (directive — write the implementation):

1. Logging: append all stdout+stderr to `$HOME/gateway-price-sync.log` with timestamped lines (mirror `prod-primary-schedule-report.sh` convention — `log()` helper prefixing an ISO-ish `date` stamp). Provide a `warn()` helper that logs but does NOT exit.

2. Dry-run + secret: read optional env `OPENROUTER_TOKEN` (script must work when unset — the OpenRouter endpoint serves unauthenticated). Read optional env `DRY_RUN` — when `DRY_RUN=1`, every `gatewayctl` mutation is echoed to the log instead of executed (so the orchestrator can preview without writing). NEVER embed the token literal in the script.

3. Numeric guard helper `is_pos_number()` — returns success only for a strictly-positive decimal (use an `awk`/regex test; reject empty, `null`, `0`, non-numeric). This is the gate that prevents writing garbage from a failed/parsed-wrong API response.

4. Gateway wrapper: a function that, given the gatewayctl args, runs `ssh -o ConnectTimeout=15 n8n-ia-vm docker exec ifix-ai-gateway /gatewayctl "$@"` — or echoes it under DRY_RUN. Do NOT wrap the remote side in `sh -c`.

5. Forex sync (independent, guarded): `curl -sS -m 15 https://open.er-api.com/v6/latest/USD` into a var; on curl failure → `warn` and SKIP forex (do not touch fx row, continue to pricing). Parse `.result` and `.rates.BRL` with `jq -r`. Only if `.result == "success"` AND `is_pos_number "$BRL"` → call `prices set-fx -usd-brl "$BRL"`. Otherwise `warn` and skip (prior fx row survives).

6. Pricing fetch (independent, guarded): `curl -sS -m 30` (with `-H "Authorization: Bearer $OPENROUTER_TOKEN"` only when the token is set) to `https://openrouter.ai/api/v1/models` into a temp file under the scratch/`mktemp`. On curl failure OR `jq` reports `.data | length == 0` → `warn` and SKIP all pricing writes (existing price rows survive); do not abort the whole script.

7. Model map: declare an editable associative array `MODEL_MAP` mapping gateway-stored model key → OpenRouter base slug. Seed it with at minimum:
   - `model.gguf` → `deepseek/deepseek-v4-flash`  (the 83%-traffic phantom)
   - `deepseek/deepseek-v4-flash-20260423` → `deepseek/deepseek-v4-flash`
   - `qwen/qwen3.5-27b-20260224` → `qwen/qwen3.5-27b`
   - `meta-llama/llama-3.1-8b-instruct` → `meta-llama/llama-3.1-8b-instruct`
   Add a comment explaining the date-suffixed gateway keys map to the undated OpenRouter `.id`, and that operators extend this map as new upstream models appear.

8. Per-model loop: for each `gateway_key → or_slug`, `jq` the cached dump for `.pricing.prompt` and `.pricing.completion` of that `or_slug`. If the slug is absent OR either value fails `is_pos_number` → `warn` and `continue` (skip that one model only — never abort). Otherwise call `prices set -model "<gateway_key>" -provider openrouter-fireworks -unit input_token -usd "<prompt>" -notes "phantom=<or_slug>, synced 260625-s3i"` and the same with `-unit output_token -usd "<completion>"`.

9. Exit 0 on a normal run even if some individual models were skipped (partial sync is acceptable; a hard error only on `set -e` from an unexpected failure). Log a one-line summary at the end (counts of fx updated / models updated / skipped).

`chmod +x` the file (set executable bit).
