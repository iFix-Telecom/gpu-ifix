# Quick 260625-k17: Dashboard Sessão Better Auth 7 dias + 2FA trust-device Summary

Sessão do dashboard `ifix-ai-dashboard` passa de 30min para 7 dias com `cookieCache.maxAge` casado ao `expiresIn`, eliminando o bounce indesejado para `/2fa/challenge` mid-session; `verifyTotp` agora marca `trustDevice` (~30d) para pular o TOTP em logins futuros no mesmo navegador.

## What Was Built

**Task 1 — Sessão 7 dias + trustDevice no 2FA** (commit `7b884a5`)

- `dashboard/src/lib/auth.ts`:
  - `session.expiresIn`: `30 * 60` → `7 * 24 * 60 * 60`.
  - `session.cookieCache.maxAge`: `1800` → `7 * 24 * 60 * 60` (casado com `expiresIn` — invariante que impede o Edge middleware de fazer fallback pessimista quando o `session_data` cookie expira mid-session).
  - `session.updateAge: 5 * 60` e `additionalFields.twoFactorVerified` mantidos inalterados.
  - Comentários inline D-15 (header + bloco session) e WR-01 reescritos para refletir a janela de 7 dias e o novo procedimento de revogação imediata: deletar a row em `public.session` no banco (a revogação não propaga às decisões do Edge middleware por até 7 dias). Referências aos runbooks (RUNBOOK-INCIDENTS.md class 4 + RUNBOOK-2FA-RECOVERY.md) preservadas, janela ajustada para 7 dias.
- `dashboard/src/app/2fa/challenge/page.tsx`:
  - `authClient.twoFactor.verifyTotp({ code })` → `verifyTotp({ code, trustDevice: true })`. O plugin twoFactor do better-auth marca o dispositivo confiável (default ~30 dias), pulando o TOTP em logins futuros nesse navegador.
- `dashboard/src/middleware.ts` deliberadamente intocado (lógica de bounce já correta; o problema era só o TTL curto).

**Task 2 — Build local do dashboard**
