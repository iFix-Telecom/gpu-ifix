# Phase quick-260625-v04 Plan 01: Tier-2 Dashboard "Operação" Summary

GET /admin/operations agrega FSM primário, agenda, lifecycles do mês, breakers e custo Vast lendo as fontes já existentes in-process; a página /operacao do dashboard renderiza os 4 painéis com polling de 10s.

## What was built

**Backend (`gateway`)**
- `Reconciler.Snapshot()` — getter exportado lockless que compõe `r.deps.FSM.State().String()` (nil-guard → "unknown"), `activeLifecycleSnapshot()` e `isLeader.Load()` num `ReconcilerSnapshot`. Sem mutex, sem escrita compartilhada (T-v04-03 mitigado).
- `admin.OperationsHandler` (`GET /admin/operations`) — espelha o padrão usage.go/metrics.go: interface `operationsQueries` isolada, construtor duplo (`NewOperationsHandler` prod + `newOperationsHandlerWithQueries` teste), envelope de erro OpenAI, counter `obs.GatewayAdminRequests` por branch. Agrega:
  - **FSM**: primary via `rec.Snapshot()` (rec nil → "unknown", ids 0, is_leader false); emerg via `fsmStateString(emergFSM)` reusado do metrics.go.
  - **Schedule**: `ParseScheduleEnv(cfg)` (puro), `ShouldBeProvisioned(now)`, `NextTransition(now)`; dias ordenados Mon→Sun como tokens curtos.
  - **Lifecycles**: janela início-do-mês em America/Sao_Paulo, `ListPrimaryLifecycles(StartedAt=monthStart, Limit=50)`; pgtype → ponteiro JSON-null (`.Valid`).
  - **Breakers**: `EffectiveStateSnapshot()` → slice ordenado por upstream.
  - **VastCost**: soma `total_cost_brl` das rows fechadas (mês/dia BRT) + accrual ao-vivo da row aberta (`accepted_dph × horas-desde-started × USDToBRLRate`); `budget_pct_used = month/budget*100` (guard div-zero). `phantom_month_brl` omitido (omitempty).
- Wiring em `main.go`: campo `adminOperationsHandler` no struct `proxies`, construção com `breakerSet`/`primaryReconciler`/`emergFSM`/`cfg`, rota `GET /operations` dentro do `adminRouter` (X-Admin-Key herdado via `.Use(admin.Middleware)` — T-v04-01 mitigado).
- `operations_test.go` — unit puro (sem DB/Docker): fake `operationsQueries` + `breaker.Set` real via miniredis; rec/emergFSM nil; asserts 200/JSON, unknown FSM, `cost_brl` null na row aberta, month/today com accrual, breakers ordenados, schedule kind∈{up,down,""} + days não-vazio, janela month-start, e 500 envelope em erro de query.

**Frontend (`dashboard`)**
- `lib/gateway.ts`: interfaces `OperationsResponse`/`OperationsFSM`/`OperationsSchedule`/`OperationsLifecycle`/`OperationsBreaker`/`OperationsVastCost` espelhando o handler Go field-for-field (nullable Go → `T | null`, phantom opcional); `fetchOperations()` via `proxyGet("operations")` (catch-all proxy já cobre, sem allowlist a mudar).
- `app-sidebar.tsx`: item "Operação" (`/operacao`, ícone `ServerCog`).
- Página `/operacao` ("use client"): `useQuery(["operations"], fetchOperations, refetchInterval=10000)`, skeleton/erro pt-BR com "Tentar novamente"; compõe os 4 painéis.
- 4 componentes: `operacao-fsm-panel` (estado primário+emerg, líder, agenda/transição traduzida), `operacao-lifecycle-timeline` (tabela; "em curso"/"rodando" para aberta), `operacao-breaker-badges` (pílulas semânticas closed=verde/half-open=âmbar/open+forced-open=vermelho), `operacao-cost-panel` (KPIs hoje/mês/budget + barra de budget com alerta ≥75%/≥90%).
