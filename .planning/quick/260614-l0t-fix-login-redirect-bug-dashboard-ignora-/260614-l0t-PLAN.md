<objective>
Fix the login redirect bug in the ai-gateway dashboard: with 2FA enabled, the
backend returns HTTP 200 `{twoFactorRedirect:true}` (no real session, only the
temporary `two_factor` cookie). The login page ignores this flag and does
`router.push("/")`, which the middleware bounces to `/login?session_expired=1`.
The `/2fa/challenge` page is never reached.

Purpose: Restore the documented better-auth `twoFactorClient` flow so 2FA users
can actually complete login.
Output: A one-branch fix in `login/page.tsx` plus a regression test pinning the
three redirect outcomes.
</objective>
