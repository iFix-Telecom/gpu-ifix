<objective>
Fix the 2FA bounce loop on the ai-gateway dashboard. The Edge middleware reads `twoFactorVerified` from Better Auth's `cookieCache` (maxAge 60s), but nothing reheats that cache during normal dashboard use — `useSession`/get-session only fires at first-login, and the overview poll hits `/api/gateway`, not `/api/auth/get-session`. After 60s the `session_data` cookieCache expires, `getCookieCache` returns null, the middleware's pessimistic fallback returns `twoFactorVerified:false`, and direct navigation loops to `/2fa/challenge`.

LOCKED FIX (user-approved, Option A): raise `session.cookieCache.maxAge` from `60` to `1800` so the cache lives the entire 30-min idle session window (`session.expiresIn = 30 * 60`). `twoFactorVerified` is monotonic within a session (false→true, never reverts), so 30-min staleness on that claim is harmless. The only tradeoff: session/2FA-revocation propagation to the Edge middleware grows from up to 60s to up to 30min — acceptable for a 4-admin internal panel.

Purpose: Stop authenticated admins from being kicked to the TOTP challenge mid-session.
Output: maxAge bump + comment/runbook wording corrected to 1800s/30min.
</objective>
