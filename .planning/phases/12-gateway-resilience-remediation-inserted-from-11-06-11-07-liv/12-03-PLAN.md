<objective>
RES-13: close the 100×502 window from the 11-07 chaos. Today the dispatcher only falls back to tier-1 when the breaker is OPEN; a connection-class dial failure (connection-refused, no route, DNS, dial timeout) with the breaker still CLOSED writes a 502 `upstream_unreachable` directly via the ReverseProxy `ErrorHandler`. This plan intercepts the dial failure at the `RoundTrip` level (BEFORE any byte is written — Pitfall 2), suppresses the default 502 in a custom `ErrorHandler`, carries the fallthrough signal back to the dispatcher via a request-scoped `dispatchResult`, and on a pre-byte connection-class error re-dispatches into the existing `tier_priority` ASC cascade. Sensitive tenants still 503 (D-10, LOCKED). Response-timeout/5xx are untouched (D-06; Phase 3 no-retry-on-tool-calls preserved).

Purpose: This is the request-path half of autonomous failover. Combined with Plan 02's D-04 force-open, the bridge (instant fallthrough) plus the stable state (breaker opens after N dials, D-09) means a dead tier-0 produces zero connection-class 502 — the D-18 chaos gate.

Output: A dial-failure-aware RoundTripper, a sentinel-suppressing ErrorHandler, an explicit dispatchResult abstraction, dispatcher cascade re-dispatch, safe GetBody-based body replay across every cascade hop, sensitive-block invariant preserved, and fallthrough metrics.
</objective>

httputil.ReverseProxy semantics (load-bearing for Pitfall 2): ServeHTTP does NOT return the transport error to the caller. When Transport.RoundTrip returns an error, ReverseProxy invokes p.ErrorHandler(w, r, err) and returns. The DEFAULT ErrorHandler writes a 502 and logs.
