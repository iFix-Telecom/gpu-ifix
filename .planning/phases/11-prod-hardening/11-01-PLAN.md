    Use psycopg double-`with` context-manager pattern with `connect_timeout=10`. SELECT body source MUST JOIN audit_log_content (reviews HIGH — the prompt JSONB lives there, not in audit_log):

    ```sql
    SELECT
      a.request_id,
      a.ts,
      a.tenant_id,
      t.slug AS tenant_slug,
      a.route,
      a.upstream,
      a.status_code,
      a.data_class,
      c.prompt AS body
    FROM ai_gateway.audit_log a
    LEFT JOIN ai_gateway.audit_log_content c
      ON c.request_id = a.request_id AND c.ts = a.ts
    LEFT JOIN ai_gateway.tenants t
      ON t.id = a.tenant_id
    WHERE a.ts >= %s AND a.ts < %s
      AND (%s OR a.data_class != 'sensitive')
    ORDER BY a.ts ASC;
    ```

    (If `ai_gateway.tenants` is not the canonical tenants table on the deployed schema, executor verifies via `\dt ai_gateway.*` first and substitutes the correct table — e.g. some schemas use `ai_gateway.tenants_admin`. Tenant-slug lookup is critical so the fixture downstream consumer can compute `IFIX_KEY_${TENANT_SLUG^^}` at replay time.)

    For each row: compute `_replay_delay_s` = ts - prev_ts (seconds). Call `sanitize_body(body, data_class, route)` which:
    - Returns None when data_class=sensitive (skip row entirely);
    - Replaces `tool_calls[].function.arguments` with `{"_replay_placeholder": true}` if present;
    - Drops `audio.filename` / `file.url` keys;
    - For route `/v1/audio/transcriptions`: returns the sanitized prompt JSONB MINUS any base64 audio bytes, AND records a marker `{"_replay_audio_part": "stub"}` so the replay engine knows to attach the stub OGG file at runtime.

    DO NOT emit: `tenant_id` (UUID; tenant_slug suffices), `_replay_api_key` (forbidden), `body` (use `_sanitized_body`), any header/cookie/authorization field.

    Apply Pattern A secret-once: NEVER log DSN; NEVER log keys. Apply 8-section spine per Pattern B (Module docstring → Constants → Config+CLI → Helpers → Audit-DB → Gates+exit → Orchestration → main). Exit codes 0/1/2 (0=ok, 1=runtime, 2=usage).
