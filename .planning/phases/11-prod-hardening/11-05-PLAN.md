  <pitfall name="DIRECT-SQL-PROBING-REJECTED">
    Direct SQL probe (psql SELECT) followed by HTTP signup-API write is REJECTED for three reasons:
    1. RACE: between the probe and the HTTP signup, a concurrent operator run can insert the same email — both runs see "not found" and both attempt signup, causing one to fail or duplicate.
    2. SCHEMA DRIFT: Better Auth owns the auth schema. A direct SQL probe against `dashboard_auth.user` can query columns that Better Auth may rename in a minor version (e.g., `email_verified` vs `emailVerified` or new join tables for invitation tokens). The probe drifts from the API contract.
    3. TWO-PATH AUTHORIZATION SURFACE: mixing psql (DB credential) and HTTP signup (no auth or anonymous) widens the secret surface — operator now needs BOTH DASHBOARD_DATABASE_URL AND HTTP access. Single-path reduces.
    Mitigation: this script uses ONE method per run — Better Auth admin invite/reset API (preferred) OR HTTP signup + HTTP GET verification (fallback). NEVER both.
  </pitfall>
