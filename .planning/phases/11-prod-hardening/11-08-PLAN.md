  <action>
    Create scripts/chaos/openrouter-iptables-drop.sh per Pattern E. The scope is **container netns ONLY** (NOT host-wide) — this is the consensus #1 requirement.

    Required structure:
    1. Shebang + set -euo pipefail.
    2. log() function to stderr with ISO-8601 + tag [openrouter-iptables-drop].
    3. Usage: ACTION="${1:?usage: $0 apply|cleanup}"
    4. Constants:
       - DOMAIN="openrouter.ai"
       - CONTAINER="ifix-ai-gateway" (override via env CHAOS_CONTAINER_NAME)
    5. Resolve container netns pid up-front: `CONTAINER_PID=$(docker inspect -f '{{.State.Pid}}' "$CONTAINER")`. If empty or non-numeric, log "container $CONTAINER not running on this host" exit 1.
    6. ipt() helper: `ipt() { sudo nsenter -t "$CONTAINER_PID" -n iptables "$@"; }` — every iptables invocation goes through this so all rules live inside the container's network namespace.
    7. apply branch:
       - Re-resolve OpenRouter IPs at runtime from n8n-ia-vm (this script runs on n8n-ia-vm so dig output reflects the gateway's view of DNS): `IPS=$(dig +short "$DOMAIN" | sort -u)`. If empty log "no IPs resolved for $DOMAIN" exit 1.
       - For each resolved IP: `ipt -I OUTPUT 1 -d "$ip" -p tcp --dport 443 -j DROP -m comment --comment "phase11-chaos-openrouter"`
       - Append 2 broad CF CIDR DROPs as rotation insurance (scoped to the container netns — host egress for other CF-fronted services stays intact):
         `ipt -I OUTPUT 1 -d 104.18.0.0/15 -p tcp --dport 443 -j DROP -m comment --comment "phase11-chaos-openrouter"`
         `ipt -I OUTPUT 1 -d 172.64.0.0/13 -p tcp --dport 443 -j DROP -m comment --comment "phase11-chaos-openrouter"`
       - Log "applied N+2 DROP rules in container=$CONTAINER netns pid=$CONTAINER_PID tagged phase11-chaos-openrouter".
       - Watch loop (optional, gated by env CHAOS_REREVOLVE_LOOP=1): every 30s for the duration of the chaos window re-run `dig +short` and insert any new IPs not yet covered (handles CF mid-test rotation). Document in docstring; default off.
    8. cleanup branch:
       - Loop: `while ipt -S OUTPUT | grep -q "phase11-chaos-openrouter"; do RULE=$(ipt -S OUTPUT | grep -m1 "phase11-chaos-openrouter" | sed 's/^-A/-D/'); ipt $RULE; done`
       - Verify inside container netns: `ipt -L OUTPUT --line-numbers | grep -c phase11-chaos-openrouter` — expect 0; log "all phase11-chaos-openrouter rules removed from container netns".
       - Defensive host check: `sudo iptables -S OUTPUT | grep -c phase11-chaos-openrouter` — MUST also be 0 (the rules were never applied host-side but defensive check guards against operator error or fallback usage).
       - If either residual count != 0, log "WARN: residual rules; manual cleanup needed" exit 1.
    9. Rollback contract (printed in docstring AND in stdout summary): if container restarts mid-test, its netns is destroyed and rules vanish naturally. If gateway behaviour after cleanup is anomalous, operator runs `docker restart ifix-ai-gateway` and re-verifies host iptables -L OUTPUT is empty of the tag.
    10. Final summary block (Pattern E) to stdout: action taken + rule count delta + container_pid recorded.

    Invocation pattern (document in script docstring): the script MUST run **on n8n-ia-vm** (where the ai-gateway container's netns lives). From ops-claude: `ssh n8n-ia-vm 'sudo bash -s -- apply' < scripts/chaos/openrouter-iptables-drop.sh`. Alternatively `scp` the script then `ssh n8n-ia-vm 'sudo bash /tmp/openrouter-iptables-drop.sh apply'`.

    Document explicitly in the docstring: this script DOES NOT modify the host-level OUTPUT chain. Other containers on n8n-ia-vm (n8n, IA workers) keep full egress to OpenRouter IPs throughout the test.
  </action>
