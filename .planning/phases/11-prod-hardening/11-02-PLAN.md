<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → /api/auth/* | unauthenticated POSTs for sign-in/sign-up/verify-totp |
| Browser → /2fa/* | authenticated-but-unverified session boundary |
| dashboard container → DO Postgres ai_dashboard_prod | TOTP secrets at rest (encrypted by plugin) |
| Edge runtime middleware → session cookie cache | twoFactor claims must be cookie-readable (no DB call in Edge) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-AUTH-01 | Spoofing | /sign-in/email credential stuffing | mitigate | rateLimit customRules `/sign-in/email: { window: 900, max: 5 }` per-IP; generic error message "E-mail ou senha inválidos" (UI-SPEC line 183) |
| T-11-AUTH-02 | Tampering | TOTP code replay within window | mitigate | better-auth twoFactor plugin auto-stores last-used counter and rejects re-use (verified node_modules/better-auth/dist/plugins/two-factor/totp/index.mjs) |
| T-11-AUTH-03 | Spoofing | TOTP brute-force on /two-factor/verify-totp | mitigate | rateLimit customRules `/two-factor/verify-totp: { window: 60, max: 5 }` |
| T-11-AUTH-04 | Spoofing | Email allowlist bypass via signUp | mitigate | databaseHooks.user.create.before runs server-side; rejects non-`@ifixtelecom.com.br` domains; D-13 |
| T-11-AUTH-05 | Tampering | Backup code re-use | mitigate | better-auth plugin auto-consumes each code one-time (verified plugin schema) |
| T-11-AUTH-06 | Spoofing | Session fixation post-2FA | mitigate | better-auth regenerates session ID on signIn + twoFactor verify endpoints |
| T-11-AUTH-07 | Information Disclosure | TOTP secret leak via response logs | mitigate | secret rendered ONCE on enroll step 1; never logged; QR + secret displayed client-side only |
| T-11-AUTH-08 | Denial of Service | rateLimit exhaustion locks admins out | accept | 4 admins total; lockout window 15min; documented in RUNBOOK-INCIDENTS class 4; 2FA recovery path documented in RUNBOOK-2FA-RECOVERY (11-09) |
| T-11-AUTH-09 | Spoofing | middleware redirect-loop if cookie claim missing | mitigate | Cookie-claim contract enforced via auth.ts cookieCache + additionalFields configuration; middleware.test.ts asserts no-loop behaviour; staging smoke covers enroll→challenge round-trip end-to-end |
| T-11-SC | Tampering | npm/pip/cargo installs | mitigate | Pre-install slopcheck audit (Task 11-02-04 Step 1) for every package that survived the Step-0 inventory; all must return `[OK]` before install runs; commit results to plan evidence |
