## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-05-01 | Tampering | Accidental force-push to `main` overwriting commits | mitigate | Verify step asserts no `--force` flags exist in the script body; script uses ONLY `git merge --ff-only` and plain `git push origin main`. |
| T-10-05-02 | Tampering | Tag created against wrong commit (operator on stale develop) | mitigate | Pre-merge `git checkout develop && git pull --ff-only` enforces fresh local develop; FF-only merge into main prevents tag landing on a diverged ancestry. |
| T-10-05-03 | Repudiation | Tag pushed without `-a` (lightweight) loses metadata / signing info | mitigate | Script hardcodes `git tag -a $SIGN "${RELEASE_TAG}" -m "..."`; verify step greps for `git tag -a`. |
| T-10-05-04 | Information disclosure | RELEASE_MESSAGE leaked into terminal scrollback / CI logs | accept | Message is intentionally public (release notes); contains no secrets. |
| T-10-05-05 | Denial of service | GHA tag-build fails after tag push; image not available for Plan 10-06 | mitigate | Script `gh run watch` blocks until build completes; on failure, operator decides whether to delete tag (`git tag -d v1.0.0 && git push origin :refs/tags/v1.0.0`) and retry — manual operator step, NOT auto-destroyed by this script. |
| T-10-05-06 | Tampering | Sentry release tag mismatch (Pitfall 5) — binary built without `GATEWAY_VERSION=v1.0.0` | mitigate | Script enforces tag-before-anything (no `docker build` happens here at all; image is built BY GHA from `refs/tags/v1.0.0` so `compute-tags` is guaranteed to set `gateway_version=v1.0.0`). |
| T-10-05-SC | Tampering | Supply-chain — npm/pip/cargo installs | accept | No package operations; pure git + gh CLI + docker pull. |
</threat_model>
