### Build Smoke — Deferred to CI

`docker buildx build --target final` was **not** run as part of this plan:

- ops-claude (worktree host) has `docker 20.10.24+dfsg1` without the `buildx` plugin.
- vps-ifix-vm has buildx v0.33.0 but running a fresh multi-stage build there would pull multi-GB CUDA base layers, occupy ~15–20 min of wall time, and push a throwaway `pod-primary:11.1-shrink-test` tag into the dev registry cache without independent value.
- Plan 11.1-03 action step 8 explicitly permits the alternative: "OR defer to CI: push a branch and confirm `.github/workflows/build-primary-pod.yml` runs green."
- The CI workflow triggers on push to `develop` filtered to `pod/primary/**` + the workflow file itself — Plan 04+ that merges this work will exercise the full multi-stage build canonically.
- The static checks above (grep audits + Dockerfile FROM/COPY --from cross-ref + supervisord configparser parse + onstart bash -n) validate every Mudança 1/2/3/8 acceptance criterion that does NOT require an actual built image.
