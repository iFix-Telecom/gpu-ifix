## Deviations from Plan

### Scope Boundary (per execute-plan Rule 3)

**1. [Rule 3 - SCOPE] Left `local-stt` references in shed/inflight_test.go + shed/subscribe_test.go untouched**
- **Found during:** Task 1 acceptance grep
- **Issue:** `gateway/internal/shed/inflight_test.go` (lines 14/117/125/136) and `gateway/internal/shed/subscribe_test.go` (line 36) use `"local-stt"` as a placeholder upstream string in registry-of-strings tests. The `InflightRegistry` and the shed `Subscriber` are role-agnostic data structures — they take any upstream name and track per-string counts/state. The literal `"local-stt"` here is not coupled to the gateway's drain logic; it's just a test fixture identifier.
- **Decision:** Out of scope. Modifying these tests would touch the InflightRegistry / Subscriber semantics — pre-existing test coverage that isn't directly affected by Phase 11.1's primary-reconciler refactor.
- **Future:** When Plan 02 deletes the `local-stt` row from `ai_gateway.upstreams`, the shed Subscriber's `Rebuild` will simply stop seeing the name in its input slice and the test fixture should be updated then (or in Plan 03 if it touches shed).

### Auto-fixes (per execute-plan Rule 1/3)

**2. [Rule 1 - Bug] Deprecated-alias `LookupEnv` semantics**
- **Found during:** Task 3 first test run
- **Issue:** Initial implementation used `os.LookupEnv` to detect "legacy var set"; this returned `true` for `t.Setenv("PRIMARY_GPU_NAME", "")` calls in `clearPhase11_1`, causing the alias path to fire on empty strings AND blocking the new var when both were set to "" by the clear helper.
- **Fix:** Replaced `LookupEnv` with `os.Getenv() != ""` checks for both the legacy and new var detections in the alias resolution block. Documented the choice inline: "operators clear vars by blanking them in Portainer, not by unsetting the entry entirely".
- **Files modified:** `gateway/internal/config/config.go`
- **Commit:** included in 0f73659

**3. [Rule 1 - Bug] Plan referenced `provisionLifecycle` call site as "line 769 evaluateAsleep"**
- **Found during:** Task 2 read-first
- **Issue:** Plan said "around line 769 evaluateAsleep" but the SearchOffers call lives in `provisionLifecycle` (line 769 was correct, function name was not). `evaluateAsleep` only delegates to `spawnProvisioning` → goroutine → `provisionLifecycle`.
- **Fix:** Modified `provisionLifecycle` (the correct function). Test `TestReconcilerVastFallback` drives `provisionLifecycle` directly (unexported but same package) with a short ctx timeout to bail out of `waitForReadyOrDestroy` after the SearchOffers + CreateInstance calls complete.
- **Files modified:** `gateway/internal/primary/reconciler.go`, `gateway/internal/primary/reconciler_test.go`

**4. [Rule 3 - Blocking] Plan referenced `primary_helpers_test.go:84` for fakeVastPrimary fixture**
- **Found during:** Task 2 read-first
- **Issue:** No `primary_helpers_test.go` file exists in `gateway/internal/primary/`. All fakes (fakeVast, fakeLoader, fakeInflight, fakeDCGMScraper) live in `reconciler_test.go` lines 64-216.
- **Fix:** Used existing `fakeVast` from reconciler_test.go directly. New test added inline next to `contains` helper.
