# Phase 11.1 Plan 00 Summary

**Wave 0 gate closed via operator-led shape pivot: 4090 fallback unviable in EU at original cap, replaced with 2×3090 fallback at $0.60/h.**

## What happened

Survey ran direct against Vast API (`GET /bundles/?q=...` URL-encoded) because `pod/scripts/vast-ai.sh search` has two bugs: (1) curl rejects nested braces in URL, (2) Vast API now demands dict form (`{eq: ...}`) for primitives that the script still sends raw. Both anomalies recorded in EVIDENCE-00, deferred to a follow-up plan (not blocking 11.1).

Primary shape 1×RTX 3090 @ ≤$0.30/h returned **10 EU offers** (cheapest CZ at $0.218/h) — gate PASS.

Fallback shape 1×RTX 4090 @ ≤$0.40/h returned **0 offers globally** — gate FAIL. Probed RESEARCH Pitfall #6 mitigations: inet floor 100 Mbps still 0, cap bump $0.50 → 1 global / 0 EU, $0.55 → 2 / 1, $0.60 → 16 / 11. Only $0.60 cap clears the ≥3 EU gate.

Operator surveyed alternate shapes: 2×RTX 3090 @ $0.60/h returned **7 EU offers** (cheapest ES at $0.483/h, average $0.42). Same GPU model as primary → single CDI/driver matrix, simpler ops. Cheaper than 1×4090 fallback ($0.42 vs $0.53 cheapest EU).

Decision: pivot fallback to 2×3090. T-11.1-04 (4090 fleet scarcity) marked RESOLVED.

## Downstream impact

Plan 01 (Wave 1) amended in-place:
- `DefaultSearchFilters` signature: `numGPUs int` → `primaryNumGPUs, fallbackNumGPUs int`
- Config gains `PrimaryVastNumGPUsPrimary` (default 1) + `PrimaryVastNumGPUsFallback` (default 2)
- `PrimaryVastPriceCapFallback` default 0.40 → 0.60
- `PrimaryVastGPUNameFallback` default "RTX 4090" → "RTX 3090"
- Legacy `PRIMARY_NUM_GPUS` → reads as Fallback alias (was multi-GPU env per memory `primary-gpu-shape-06.8-final`)

CONTEXT.md D-A6 paragraph and env var checklist (touchpoint #14) updated. Memory note `primary-gpu-shape-06.8-final` partially preserved — 2×3090 fallback keeps allowlist 43803/55158 hosts as candidates.
