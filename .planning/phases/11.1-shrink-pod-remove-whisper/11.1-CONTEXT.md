<decisions>
## Implementation Decisions

### Pre-flight & ROI Gate
- **D-A1:** **Skip audit STT prod 7d.** SEED promotion checklist sugere "auditar volume antes" mas ROI é óbvio: Vast savings $216-288/mês (cap $0.60→$0.30 contínuo) vs custo OpenAI whisper-1 $0.006/min — break-even ~30min STT contínuo/h é absurdo p/ workflow batch raras. Decisão sem audit; capturar volume real retro post-deploy se quiser validar economia exata.

### Scope Bundle
- **D-A2:** **Bundle Infinity venv removal** no mesmo PR. Phase 06.7 D-03 rollback dead-code (~600MB pip install infinity-emb + venv + supervisord block). Touchpoints simétricos ao Speaches cleanup. Imagem -600MB extra. Plan dedicated dentro do phase.
- **D-A3:** **Chatterbox TTS preservado, fora de escopo.** SEED out-of-scope explícito. Phase 11.1 não toca `pod/primary/chatterbox_server.py`, `pod/primary/test_chatterbox_shim.py`, ou supervisord chatterbox block. Auditoria volume TTS = SEED-011 follow-up candidato.

### DB Schema Cleanup
- **D-A4:** **DELETE migration** (não soft-disable) para `ai_gateway.upstreams` row `local-stt`. Migration 0027+ remove row + qualquer FK/cascade adjacente. Roll-back path = re-apply seed migration (versão anterior). audit_log preserva `upstream_name` string-only — histórico forensics intacto sem FK.
- **D-A5:** **DELETE `model_aliases (whisper, local-stt) → Systran/faster-whisper-large-v3`** na mesma migration. **PRESERVAR explicit**: `upstreams` row `openai-whisper` (tier-1) + `model_aliases (whisper, openai-whisper) → whisper-1` — `POST /v1/audio/transcriptions {"model":"whisper"}` continua resolvendo via tier-1 OpenAI direct. **Não estamos removendo whisper-1 OpenAI, só o tier-0 local Speaches/faster-whisper-large-v3.**

### GPU Shape Post-Shrink
- **D-A6:** **PRIMARY_GPU_SHAPE = 1×RTX 3090 primary + 1×RTX 4090 fallback.** DefaultSearchFilter logic: prefer `num_gpus=1 + gpu_name=RTX_3090 + price_cap=$0.30/h`; fallback `num_gpus=1 + gpu_name=RTX_4090 + price_cap=$0.40/h` se zero qualified offers no primary filter. Captura -50% custo no caso comum (3090 abundant na Vast pool) + robustez contra scarcity (4090 sempre disponível). Cap composto efetivo $0.30-0.40/h vs atual $0.50-0.60/h.
- **Allowlist refresh:** Standing config `primary-gpu-shape-06.8-final` (allowlist 43803/55158, 2×3090) DEPRECATED por este phase. Plan deve identificar novo allowlist via `vast-offer-search-method` recipe (qualified hosts 1×3090 + 1×4090 baseline).

### Dependency
- **D-A7:** **Phase 11 deferred UATs (11-06/07/08) NÃO bloqueia 11.1.** Phase 11 STATE = `closed passed_partial`. Pod template refactor é decoupled do load/chaos test target em Phase 11. 11.1 procede paralelo enquanto corpus prod acumula para re-attempt 11-06/07/08.
