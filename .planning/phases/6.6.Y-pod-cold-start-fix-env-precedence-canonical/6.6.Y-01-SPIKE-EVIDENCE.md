## Answer Q1 — host-specific vs universal port-bind failure

**Nuanced finding — record BOTH layers:**

- **Docker/Vast-API layer: ports DID bind.** Instance reached `actual_status=running` (~22:56Z, ~19.5 min after 22:36:39Z create) with a populated `ports` map — `22/tcp→40579, 8000/tcp→40554, 8001/tcp→40556, 8003/tcp→40507, 9400/tcp→40534`, all `HostIp 0.0.0.0`. `status_msg`: "success, running ghcr.io/ifixtelecom/converseai-pr…". This DISTINGUISHES the n=2 host from a "never-binds" hypothesis.
- **External-reachability layer: NEVER reachable.** `ssh_host`/`ssh_port` stayed **null the entire lifetime**. External TCP probes from **TWO** vantage points (ops-claude 10.10.10.10 AND vps-ifix-vm 10.10.10.30) **timed out (NOT refused)** on ALL published ports — including the sshd mapping 40579 — continuously for 10+ min after `running`:
  ```
  port 40579: CLOSED/timed out
  port 40554: CLOSED/timed out
  port 40556: CLOSED/timed out
  port 40507: CLOSED/timed out
  port 40534: CLOSED/timed out
  ```
  **timeout ≠ refused** → packets dropped upstream (host firewall / NAT), not an absent listener.

**Gateway-observable readiness was NEVER achieved.** Gateway FSM stuck `provisioning`; `pod_url` empty, `pod_instance_id` empty for the full 40+ min window. The gateway emitted **ZERO reconciler lines after the offer-pick for 40+ min** — the silent empty-URLs poll branch is CONFIRMED live (this is plan 6.6.Y-03's per-strike log target). Operator force-down at 23:01:35Z → FSM asleep, Vast instances list = 0 (clean destroy).

**Verdict: failure is NOT host-specific-to-Brazil.** n=2 of 2 sampled hosts (iter-1 Brazil 348505 + iter-2 South Africa 317118) failed to become gateway-reachable. Treat as **universal-until-proven-otherwise.**

### >> DIRECTIVE for plan 6.6.Y-03 / 6.6.Y-04 (Option D — universal failure)

Because the failure is universal across sampled hosts, the Vast `ports` field is **NOT a reliable readiness signal** — Option B (runtime fail-fast) MUST gate on **gateway-observable reachability (URLs probe)**, NOT on the Vast API `ports` map.
