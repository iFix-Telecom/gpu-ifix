## Post-review addendum: CR-01 + CR-02 fixes verified

**Written:** 2026-06-11 (post code-review, post redeploy)

### Context

The initial VERIFICATION.md was written after the live UAT (plan 07) with status `passed_partial`.
A code review ([6.6.Y-REVIEW.md](./6.6.Y-REVIEW.md)) subsequently found 2 Critical issues:

- **CR-01:** Option B fail-fast gated on `buildPodURLs` (the Vast `ports` map) — the exact signal
  the spike DIRECTIVE forbade. Both observed failures had running status + populated ports map + TCP
  unreachable, so they bypassed the gate entirely and burned the full cold-start budget silently.
- **CR-02:** `wait "$QWEN_PID" "$BGE_PID" "$WHISPER_PID"` (multi-PID) returns only the last PID's
  exit status in bash, silently swallowing failed Qwen/BGE-M3 downloads and proceeding to
  `supervisord exec` with missing or corrupt weights.

### Fixes verified in codebase

**CR-01 — commit c518d6e** (`fix(6.6.Y): CR-01 — port-bind budget gates on connection-level reachability, not Vast ports map`):

- `gateway/cmd/gateway/main.go`: `primaryReachable` closure added (lines ~963–991), dials TCP via
  `net.Dialer{Timeout: 5s}`, returns `true` for connection-refused (host up, service warming) and
  `false` only for dial-timeout / no-route (host never NAT-published — the spike failure signature).
  Wired as `Deps.Reachable: primaryReachable` in `NewReconcilerFull`.
- `gateway/internal/primary/reconciler.go`: `if r.deps.Reachable != nil && !r.deps.Reachable(ctx, urls.LLM)`
  at line 1109 fires `public_port_bind_timeout` when contiguous running time exceeds `portBindBudget`.
  A `connect success/refused` (host reachable) continues polling under the cold-start budget per
  the design note invariant.
- Three new tests pass under `-race`:
  - `TestWaitForReady_RunningPortsBoundButTCPUnreachable_DestroysWithinBudget` — confirms spike
    failure class (running + ports bound + TCP-unreachable) is destroyed within budget.
  - `TestWaitForReady_ReachableButNotReady_NoFalseClose` — confirms TCP-reachable-but-not-HTTP-ready
    pods are NOT falsely destroyed.
  - `TestWaitForReady_PublicPortBindTimeout` — empty-URLs subclass (pre-existing) still works.

**CR-02 — commit 6765434** (`fix(6.6.Y): CR-02 — onstart waits each weight download PID individually`):

- `gateway/internal/primary/onstart.go` lines 167–174: each PID waited individually with
  `wait "$PID" || FAIL=1`; if `FAIL` is set, the script prints a FATAL message and `exit 1` before
  `exec supervisord`. All 3 weights (Qwen, bge-m3, whisper) now have integrity fail-fast.
