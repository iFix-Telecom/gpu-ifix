# Phase 6.6.Y Plan 02: Env-Precedence Hard Fail-Fast + Cold-Start Config Plumbing Summary

Replaced the inert Phase 11.1 soft-warn legacy-alias block with a hard fail-fast that refuses gateway boot when any legacy primary-shape env var is present (value-agnostic via `os.LookupEnv`), deleted the 3 legacy struct fields/reads, added two cold-start config readers consumed by plan 6.6.Y-03, and replaced the legacy `price_cap_dph` boot-log arg with a per-shape resolved-shape dump so operators can confirm env precedence at boot.

## What Was Built

**Task 1 — Hard fail-fast + field deletion + cold-start readers (commit 5b57dc5):**
- New sentinel `ErrLegacyPrimaryEnv` declared alongside `ErrMissingEnv` / `ErrInvalidURLSuffix`.
- The alias block (config.go old :608-642) is now a `legacyErrs []string` accumulator following the `ErrInvalidURLSuffix` idiom. For each of `PRIMARY_GPU_NAME`, `PRIMARY_VAST_PRICE_CAP_DPH`, `PRIMARY_NUM_GPUS`, presence is detected with `os.LookupEnv` (ok=true even for empty value) and a named migration hint is appended. `len>0` → `return Config{}, fmt.Errorf("%w: %s", ErrLegacyPrimaryEnv, strings.Join(...))`.
- Deleted 3 legacy struct fields (`PrimaryVastPriceCapDPH`, `PrimaryGPUName`, `PrimaryNumGPUs`) + their env reads. Kept emerg `VAST_PRICE_CAP_DPH` (different subsystem), `csvOr`, allowlist/blocklist reads.
- Added `PrimaryPublicPortBindBudgetSeconds` (`PRIMARY_PUBLIC_PORT_BIND_BUDGET_SECONDS`, default 120 per D-02) and `PrimaryVastRejectPrivateIP` (`PRIMARY_VAST_REJECT_PRIVATE_IP`, default true; opt-out only via literal `"false"`).
- config_test.go: new ErrLegacyPrimaryEnv subtests (non-empty + empty-string presence per legacy var), migration-hint `strings.Contains`, all-canonical-no-legacy positive (uses `os.Unsetenv` so `LookupEnv` ok=false), emerg-var-negative. Added `unsetLegacyPrimary` helper + wired into `clearPhase6_6` / `clearPhase11_1`.

**Task 2 — Boot-time resolved-shape log dump (commit 7597def):**
- Removed the build-breaking `"price_cap_dph", cfg.PrimaryVastPriceCapDPH` arg.
- Added `shape0_*` (PRIMARY 1×3090@0.30), `shape1_*` (FALLBACK 2×3090@0.60), `allowlist`, `port_bind_budget_seconds`, `reject_private_ip` to the `"Phase 6.6 primary reconciler started"` log.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Existing test clear-helpers blanked legacy vars → would hard-fail**
- **Found during:** Task 1
- **Issue:** `clearPhase6_6` / `clearPhase11_1` set the 3 legacy vars via `t.Setenv(v, "")`. Under the new presence-based policy, set-empty = present = `ErrLegacyPrimaryEnv`, which would have broken ~6 unrelated existing tests (`TestConfig_PrimaryPod_DefaultsLoaded`, `TestPrimaryVastShapeDefaults`, etc.).
- **Fix:** Added `unsetLegacyPrimary(t)` helper (`os.Unsetenv` + `t.Cleanup` restore) and removed the 3 legacy entries from the blank-lists; both clear-helpers now call it. The plan anticipated this ("use os.Unsetenv in this subtest").
- **Files modified:** gateway/internal/config/config_test.go
- **Commit:** 5b57dc5

**2. [Rule 3 - Blocking] Deleted-field references in two other test files broke the build**
- **Found during:** Task 1 (cross-package grep)

- **Fix:** Deleted both stale assignments (per-shape caps already set in the same helpers; the deprecated cap was redundant).
- **Files modified:** gateway/internal/primary/reconciler_test.go, gateway/internal/integration_test/primary_helpers_test.go
- **Commit:** 5b57dc5

- **Fix:** Replaced with assertions on the new cold-start readers (`PrimaryPublicPortBindBudgetSeconds` default 120 / override 200; `PrimaryVastRejectPrivateIP` default true / override false). The DPH default is now covered by `TestPrimaryVastShapeDefaults` per-shape caps.
