## Decisions Made

- **CaptureBreadcrumb prefix moved to caller.** Original method was `(*Reconciler).captureBreadcrumb(category, data)` and prepended `"emerg." + category` internally. Free-function form takes the full prefixed string. emerg sites now pass `"emerg.<event>"` explicitly so the Sentry breadcrumb body is byte-equivalent to the pre-extraction form. Wave 2 primary code will pass `"primary.<event>"`.
- **BestEffortDestroy keeps an unused ctx parameter.** The destroy mints its own background context with a 30s budget (Pitfall 8). The ctx arg is reserved for forward-compat tracing spans; explicit `_ = ctx` documents the intent.
- **destroyShutdownBudget duplicated.** Both `emerg/lifecycle.go` (used by recovery.go inline destroy) and `vastutil/helpers.go` (used by BestEffortDestroy) hold the same `30 * time.Second` const. Duplication is cheaper than re-routing recovery.go's inline destroys through vastutil and avoids touching the orphan-reconciliation path in this refactor.
- **lifecycle_test.go duplicate test removal.** The plan explicitly said "deletar tests duplicados (TestFilterBelowCap_*, TestExcludeHost, TestMustEventJSON, TestPgInt8, TestPgNumericFromFloat) — já cobertos por vastutil". Coverage moved cleanly. The remaining 15 emerg-internal tests (buildCreateRequest, podHealthURL, errReason, emergencyOnstart suites) stayed verbatim.

**2. [Rule 2 — Missing Critical] Updated callers outside `lifecycle.go` (reconciler.go, recovery.go, budget.go, budget_test.go)**

- **Found during:** Task 3 (initial build attempt after deleting helper definitions in lifecycle.go)
- **Issue:** Plan action explicitly listed `lifecycle.go` call-sites for the rewrite.
