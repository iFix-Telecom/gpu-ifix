## Decisions Made

- **sqlc output path = `internal/db/gen/`, not `internal/db/sqlc/`.** The plan body and acceptance criteria reference `internal/db/sqlc/primary_lifecycles.sql.go`, but the existing `gateway/sqlc.yaml` config specifies `out: internal/db/gen` with `package: gen`. Existing emergency_lifecycles.sql.go (and the entire sqlc-generated set) live at `internal/db/gen/`. Aligning the new file with the actual generator config preserves the package symbol path (`gen.Queries`) and keeps `models.go`/`querier.go` regen contiguous. See deviations section below — this is the only divergence from the plan's literal path string.
- **`ClosePrimaryLifecycle` is idempotent (`WHERE id = $1 AND ended_at IS NULL`).** The emerg analog `CloseEmergencyLifecycle` lacks this guard, but the plan's `<interfaces>` SQL snippet specifies the guard explicitly. Preserved literally because retry storms after the schedule down-window can issue the close twice (reconciler restart picks up an already-closed row via `GetOpenPrimaryLifecycle` returning `ErrNoRows`, but a race window remains).
- **`InsertPrimaryLifecycle :one` returns `id, started_at` (2 cols), not just `id` like emerg's analog.** The plan's `<interfaces>` SQL specifies `RETURNING id, started_at` and downstream callers (06.6-06a reconciler) need `started_at` for duration computation. Generated `InsertPrimaryLifecycleRow` struct exposes both fields.
- **PrimaryEvent has no `Payload map[string]any` field.** The plan-specified shape is 6 scalar fields (Type, State, LifecycleID, Reason, SinceUnix, ReplicaID); EmergEvent has an additional `Payload` map because emerg events carry per-Type extensions (offer_id, dph on accept). Primary lifecycle stores those IDs in the Postgres row, not on the channel — the channel is a thin notification surface. Documented in the `PrimaryEvent` doc comment.
- **TDD RED commit message describes the failure mode explicitly.** The RED commit (`bcd3668`) enumerates which symbols are undefined so a grep over commit history can find the GREEN follow-up.
