- **Found during:** Task 5 (initial test run)
- **Issue:** Initial header comment in `pod/primary/supervisord.conf` contained the literal string `--chat-template-file` (in prose form explaining what was being avoided). The structural test asserted the file does NOT contain that substring (B1 embedded LOCKED invariant), so the assertion failed.

- **Files modified:** `pod/primary/supervisord.conf`
- **Verification:** `TestSupervisordConf_4ProgramBlocks` now passes; assertion intent preserved.
- **Committed in:** `d2bb1cb` (folded into Task 5 commit; the comment text drift was incidental to Task 1 and would have been a flake otherwise).

**2. [Rule 3 - Blocking] Initial `TestPrimaryOnstart_QuotedEnvExpansions` regex was too crude — flagged `"ifix/$MINIO_BUCKET/$key"` as unquoted**
- **Found during:** Task 5 (initial test run)
- **Issue:** The first regex (`[^"]\$(MINIO|PRIMARY)_[A-Z0-9_]+[^"}]`) only inspected the immediate adjacent characters around `$VAR` and falsely flagged expansions that sit inside multi-character quoted strings like `"ifix/$MINIO_BUCKET/$key"` (the leading slash is what tripped the regex even though the entire token is enclosed in double quotes).
- **Fix:** Replaced the naive regex with a line-walking quote-balance check: for each `$MINIO_*` / `$PRIMARY_*` expansion site on a line, count unescaped `"` characters preceding it on the same line and assert the count is odd (i.e. the expansion sits inside an open quote run).
- **Files modified:** `gateway/internal/primary/lifecycle_test.go` (test logic only — no production code change).
- **Verification:** Re-run `go test ./gateway/internal/primary/... -count=1 -race` → all 28 tests green.
- **Committed in:** `d2bb1cb` (Task 5 commit).
