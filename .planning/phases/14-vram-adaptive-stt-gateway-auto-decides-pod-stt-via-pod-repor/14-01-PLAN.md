<objective>
Gateway side (SEED-019 part 3): replace the manual `PRIMARY_POD_SERVE_STT` config flag with a
per-lifecycle device boolean derived from a pod-reported `whisper_device` value, capture that value
into `primaryPodURLs` at Ready, gate the 3 reconciler stt-override sites on it, and delete the flag.

This plan ALSO carries Wave 0: it reworks the two flag-pinned tests to drive off the device signal
(RED before GREEN) so the implementation in this same plan turns them green.

Purpose: STT-from-pod becomes automatic + shape-agnostic on the gateway side. The fail-safe (unknown
device → no override → gemini-stt) preserves today's prod behavior during the pod-image rollout window.

Output: gateway compiles with `PrimaryPodServeSTT` gone; device-driven stt gate green under unit test;
the real device-report fetch is wired in main.go.

This plan defines the device-report CONTRACT (the parsed JSON field name + URL path the gateway reads)
that Plan 14-02 must produce on the pod side. Interface-first: the contract ships here.
</objective>
