<objective>
Make the existing `UsageInterceptor` WRITE the audio + embed usage atomics it already
reads at flush time, IMPLEMENTING LOCKED CONTEXT DECISION #2 IN FULL: audio seconds come
from the response `duration` field WHERE PRESENT, **ELSE derived from the request audio
length**. This matters because the gateway does NOT force `response_format=verbose_json`
(confirmed: `grep -rn verbose_json internal/ cmd/` finds nothing), so the DEFAULT
OpenAI/Speaches/Whisper transcription response is `{"text":"..."}` with NO duration field.
A producer that only reads response `duration` would meter 0 for the common-case client —
the bug would NOT be fixed and success criterion #1 would fail.

STT + embed responses are `application/json`, so they flow through `usageJSONBuffer`
(interceptor_usage.go:124), whose `Close()` runs `FinalizeRequest` — which deletes the
accountant slot via a deferred `accountant.Delete` (:197). So the JSON-buffer path parses +
Stores the atomics INSIDE Close, then the slot is gone; and the production `flusher` is a
concrete `*billing.Flusher` requiring a `*pgxpool.Pool` (flusher.go:55,64), not an
interface — so the Store is NOT observable Postgres-free through the accountant. To keep the
unit tests DB-free, the route-dispatched Store is factored into a PURE helper
`applyAudioEmbedUsage(usage, route, body, reqCtx)` that operates on a passed-in
`*RequestUsage`; Close calls it (production path unchanged), tests call it directly.

This plan adds the producers IN the existing interceptor (mirroring the chat-path token
extractor — locked decision #1: no new billing path), the request-audio derivation fallback
(a small middleware + duration helper), and proves the unit conversion + the no-duration case
with tests. Wiring the proxies + mounting the middleware is Plan 16-02 (depends on this plan).

Purpose: turn the dead audio/embed metering dimension into live data — non-zero even for
default-format STT — with a testable seam.
Output: pure Store helper + request-audio duration helper + stamping middleware + tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| upstream STT/embed → gateway interceptor | Upstream response JSON parsed for duration/data[]; oversized body must not OOM or crash close |
| client request audio → STT middleware | Untrusted multipart audio decoded for duration; malformed container must not panic or be re-forwarded corrupted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-01 | Denial of Service | usageJSONBuffer parse of upstream body | mitigate | Reuse jsonBufferCap (128 KiB) bound; oversized → source=partial, never OOM |
| T-16-02 | Tampering | upstream/client reports inflated duration to inflate billing/quota | accept | STT providers first-party/contracted; request-derived duration is bounded by the actual uploaded bytes — same trust extended to token counts |
| T-16-07 | Denial of Service | STT middleware reading + buffering request audio (one full in-RAM copy on the hot path) | mitigate | Bound the read to maxSTTBodyBuffer (25 MiB) via io.LimitReader; body restored via r.GetBody so the dispatcher replay stays bounded + byte-identical |
| T-16-08 | Tampering | malformed WAV/compressed container in DeriveAudioSeconds | mitigate | Header parse is bounds-checked; parse failure / empty input returns 0 (no panic), proven by the empty/garbage unit test |

No npm/pip/cargo installs in this plan — package legitimacy gate N/A.
</threat_model>
