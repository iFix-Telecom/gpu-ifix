proxy.ComposeInterceptors(ics ...ProxyResponseInterceptor) func(*http.Response) error
  — order-preserving; first arg runs first.

GEMINI ORDERING (W-1 — make explicit so the executor can't "simplify" it wrong): gemini's
existing modifyResponse FLATTENS candidates→{"text":...} AND sets Content-Type=application/json
+ mutates the body. The usage interceptor must see the FLATTENED body. So wire gemini's
ModifyResponse as an EXPLICIT closure: call modifyResponse(resp) first; if it returns nil,
THEN call usageInterceptor.Intercept(resp). Do NOT collapse this into a bare
`ComposeInterceptors(flattenAsInterceptor, usageInterceptor)` — the flatten is a
func(*http.Response) error, not a ProxyResponseInterceptor, and its body/Content-Type
mutation must complete before Intercept reads them. (groq/openai-whisper/openai-embed have NO
existing ModifyResponse, so `ComposeInterceptors(usageInterceptor)` is correct for those three.)

REQUEST-AUDIO MIDDLEWARE MOUNT: Plan 16-01 adds proxy.RequestAudioSecondsMiddleware(log). The
audit middleware is mounted on the authed /v1/* group at main.go:1380 (pg.Use(audit.Middleware(...)));
the transcriptions route is mounted at the /v1/audio/transcriptions handler (near the
/v1/embeddings mount :1437). Mount RequestAudioSecondsMiddleware so it runs BEFORE the STT
proxy on the transcriptions route (either group-level Use alongside audit, or wrapping the
transcriptions handler). It is a no-op on non-audio routes, so group-level Use is safe.

quota/counters.go CheckQuotaToday/CheckQuotaMonth + the countersQueries interface (stubbable
without Postgres — see existing counters_test.go for the fake) is where the trip-test lives.
