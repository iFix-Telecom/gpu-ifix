# Phase 06.8 Plan 04: GPU Topology Fallback Runbook Summary

Operator runbook + per-shape env presets encoding the GPU fallback topology ladder (2×3090 → 5090 → deferred split shape), where each shape is a set of `PRIMARY_*` env values and a shape switch is a manual env swap + `gatewayctl primary force-down`/`force-up` — chosen over a config-driven orchestrator because the gateway is single-primary and the reconciler's allowlist→broaden + `FilterBelowCap` + cooldown already cover intra-shape resilience.

## What Was Built

`docs/RUNBOOK-GPU-TOPOLOGY-FALLBACK.md` (mirrors the structure/voice of `docs/RUNBOOK-PRIMARY-POD-TTS.md`), with sections:

1. **Why a runbook, not an orchestrator** — cites RESEARCH: single-primary; broaden-search + cooldown handle intra-shape; a config-driven N-shape FSM is DEFERRED to SEED-002.
2. **Shape preset table** — columns `PRIMARY_GPU_NAME`, `PRIMARY_NUM_GPUS`, `PRIMARY_VAST_PRICE_CAP_DPH`, allowlist seed, approx cost:
   - Shape A (preferred) 2×RTX 3090 — `RTX 3090` / `2` / `0.60` / allowlist `43803` / ~$0.29–0.40/h
   - Shape B (fallback) 1×RTX 5090 — `RTX 5090` / `1` / `2.20` / empty / ~$0.74–2.00/h (the gateway coded defaults)
   - Shape C (degraded) — DEFERRED placeholder
3. **Transition procedure** — env swap (Portainer/.env) → `force-down` (if old-shape pod up) → `force-up`; note that `force-up` bypasses cooldown.
4. **When to fall back** — signal = repeated `provisionLifecycle` close with `no_offers_below_cap` despite a manual `force-up`; plus the Pitfall-3 / A2 search-gate caveat (validate offers actually return through the `cuda_max_good`/`driver_vers` gates before concluding the shape is dry).
5. **Image** — all shapes boot the same `PRIMARY_TEMPLATE_IMAGE`; TODO placeholder to point it at the promoted STT-fix image digest (Plan 02) once available.
6. **Shape C** — explicitly DEFERRED (depends on dual-primary routing, also Deferred per CONTEXT).
7. **Spend guard** — `MONTHLY_PRIMARY_BUDGET_BRL` (default 800.0) backstops every shape; per-shape `PRICE_CAP` + `force-down` are the controls (threat T-06.8-09).
+ **Amend-later checklist** — validated machine_ids (Plan 03 `06.8-GW-2GPU-LIVE-UAT.md`) + STT-fix image digest (Plan 02 `06.8-STT-LIVE-VALIDATION.md`), both flagged as not-yet-existing wave-2/3 outputs.

Verified against config defaults read from `gateway/internal/config/config.go` (GPU_NAME `"RTX 5090"`, NUM_GPUS `1`, PRICE_CAP `2.20`, ALLOWLIST empty, BUDGET `800.0`, TEMPLATE_IMAGE `llama.cpp:server-cuda-b9191@sha256:cb375311…`).
