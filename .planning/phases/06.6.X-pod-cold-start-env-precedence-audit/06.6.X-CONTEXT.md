## Implementation Decisions

### Repro Method (Cold-start)

- **D-01:** Spike vivo em pod fresco, NÃO forensics passiva. Provisionar via stack `ai-gateway-dev` em vps-ifix-vm (path real do FSM gateway, mesmo código que rodou 11.2 UAT). Trace markers no onstart + captura tempo-real de Vast events + supervisord + ssh-into-pod.
- **D-02:** 2 iterações máximo, cap orçamentário Vast ~$1.20 (2-4h pod 2×3090). 1ª iter = baseline com estado atual (confirmar repro). 2ª iter = variante (ex: weights mockados/baked) pra isolar causa. Parar antes se root cause já confirmado.
- **D-03:** Provisionamento via gateway live-tick (NÃO vast-ai.sh manual nem UI Vast.ai). Reproduz path real incluindo qualquer drift env entre o que código lê e o que MEMORY documenta.

### Sinais Instrumentados (4 famílias, todas obrigatórias 1ª iter)

- **D-04:** Vast-side — polling `GET /instances/<id>` a cada 5s; registrar timeline `actual_status` + `intended_status` + `status_msg` + `port mapping`. Confirma quando/onde transição para.
- **D-05:** Pod-side — `ssh root@<pod>`: tail `/var/log/supervisord.log` + `/var/log/onstart.log` + `dmesg` + `ps aux` + `df -h /weights`. Confirma onstart terminou, supervisord subiu, weights presentes em `/weights/{qwen,whisper,bge-m3}`.
- **D-06:** Gateway-side — `docker logs ai-gateway-dev --since=<provision-ts>` + `SELECT * FROM primary_pod_lifecycle ORDER BY id DESC LIMIT 1`. Confirma FSM decisões + quando `waitForReadyOrDestroy` timeouta + `terminalStrikes`/`notFoundStrikes` counters.
- **D-07:** Network — loop externo `nc -zv <pod-ip> <mapped-port>` nos 4 ports (llama 8080, speaches 5000, embed 8000, dcgm 9400) a cada 10s. Confirma quando port mapping aparece + qual serviço sobe primeiro.

### Escopo Diagnóstico Cold-start

- **D-08:** Árvore aberta com priorização por evidência — NÃO hipótese-única. Lista de suspeitos a instrumentar na 1ª iter (todos 4):
  - Onstart exit code + tempo (terminou? errou? quanto demorou?)
  - Weights presence (`/weights/qwen` ~24GB, `/weights/whisper` populado, `/weights/bge-m3` extraído + sha256 amostra)
  - Supervisord boot order + per-program status (`supervisorctl status` + per-program `startsecs`/`exitstatus`)
  - Vast port mapping + sshd reachability (Phase 06.6 sshd-first fix regrediu?)
- **D-09:** 2ª iter testa hipótese top-1 ranqueada pela evidência da 1ª (ex: se `/weights/whisper` vazio + supervisord speaches em restart loop → testar pre-bake ou onstart-3-way-download).

### Env Precedence Audit

- **D-10:** Auditoria tripla: código + runtime real + docs.
  - **Código:** grep todas vars `PRIMARY_*GPU*`, `PRIMARY_*PRICE*`, `PRIMARY_*ALLOWLIST*` em `gateway/` `pod/` — mapear default, ler-com-warn (legacy alias), readers (`config.go`), consumers (search params, lifecycle).

  - **Docs:** `RUNBOOK-OPS.md` + `RUNBOOK-PRIMARY-POD.md` + MEMORY `primary-gpu-shape-06.8-final` + `gateway/.env.prod.example` + `gateway/.env.example`.
- **D-11:** Entregar tabela: `var → valor-runtime → valor-prod-example → valor-runbook-doc → onde-lido-no-código → divergência? (Y/N) → impacto`.
- **D-12:** Escopo restrito a **shape vars apenas**: `*NUM_GPUS*`, `*GPU_NAME*`, `*PRICE_CAP*`, `*ALLOWLIST*`. Não auditar TODAS PRIMARY_* (~30 vars) — carry-forward só menciona drift de shape; outras vars ficam pra phase dedicada se surgir drift.

- **MinIO `mc` alias `ifix`** — `~/.claude/CLAUDE.md` documents endpoint + creds; spike can mock-replace whisper weight key to test the "weights missing" hypothesis without rebuilding image.

### Established Patterns

- **Onstart 3-stage trace markers** (`pod/primary/onstart.go`) — log "spawning", "waiting", "extracting". Spike instrumentation extends this pattern (no new format).
- **Legacy alias warn-on-set** (`config.go:622-642`) — `slog.Warn` when legacy var set + new slot empty. Audit must propose: keep pattern, change mapping target (Primary vs Fallback), or hard-fail.
- **Vast.ai search offer-search method** — MEMORY `vast-offer-search-method.md` documents qualified-offer query recipe; reuse for spike pod selection.
- **`waitForReadyOrDestroy` cold-start budget** — `PrimaryProvisionColdStartBudgetSeconds` env var caps total time. Spike must record this default vs runtime value (potential 3rd drift point not in carry-forward).

### Integration Points

- **Gateway → Vast.ai REST API** (instance create/get/destroy). Spike consumes same path as live-tick — zero code changes needed for repro.
- **Gateway → primary_pod_lifecycle DB writes** (`closeLifecycle` calls). Spike reads these post-hoc; no code changes.
- **Pod → MinIO** (onstart `mc share download` → curl). If weights-missing root cause, fix likely lands here (3rd download URL).

- **Operator → Portainer stack.env / `.env`** — env precedence audit reads here; fix may land in alias mapping (`config.go`) OR in docs only (no code change if mapping is correct + operator instruction wrong).
