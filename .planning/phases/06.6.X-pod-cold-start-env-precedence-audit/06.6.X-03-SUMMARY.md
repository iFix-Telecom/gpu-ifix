# Phase 06.6.X Plan 03: Docs-side Shape Var Audit — Summary

Docs-side shape-var capture (5 sources × 7 vars) for the env-precedence audit feeding plan 09; surfaced 7 documentation gaps including a missing runbook file and a code-vs-runbook PRICE_CAP divergence within a single source.

## What Was Built

Single intermediate artifact `06.6.X-AUDIT-DOCS-VALUES.md` containing:

- Source inventory (5 doc paths + existence status)
- 7-row × 5-source matrix (each cell = value+file:line OR `NOT MENTIONED` OR `FILE NOT FOUND`)
- Raw grep output per source for audit trail
- 7-item "Documentation gaps" prose section feeding plan 09 recommendations
- Plan 09 hand-off paragraph describing how to consume the doc

## Method

Per plan 03 `<action>`, ran `grep -nE 'PRIMARY_.*(NUM_GPUS|GPU_NAME|PRICE|ALLOWLIST)'` against each source. Followed up with broader sweeps (`num_gpus|gpu_name|allowlist|RTX 3090|RTX 4090|machine_allowlist`) over RUNBOOK-PRIMARY-POD.md and `.env.example` to confirm strict-regex negatives weren't hiding loose mentions. No inference: any source/var combination not returning a grep hit became literal `NOT MENTIONED`.

## Key Findings (preview for plan 09)

1. **`PRIMARY_VAST_PRICE_CAP_DPH` carries 2 different values across the doc family** — `0.40` (RUNBOOK-PRIMARY-POD.md:330,875, RTX 4090 era) vs `0.60` (.env.prod.example:120 + MEMORY:18, RTX 3090 standing). Single source-family divergence regardless of runtime (plan 02 territory).
2. **`PRIMARY_NUM_GPUS`, `PRIMARY_GPU_NAME`, `PRIMARY_VAST_MACHINE_ALLOWLIST` are absent from RUNBOOK-PRIMARY-POD.md.** Runbook gives operators no way to set the shape vars that the 11.2 Item 3 drift hinged on.
3. **New canonical names (`PRIMARY_VAST_NUM_GPUS_PRIMARY/FALLBACK`, `PRIMARY_VAST_GPU_NAME`) are NOT MENTIONED in any of the 4 existing doc sources.** Every doc still uses the legacy aliases that the code at `config.go:622-642` (per CONTEXT.canonical_refs) is warning about.
4. **`gateway/docs/RUNBOOK-OPS.md` does not exist** — plan 03 `<read_first>` references it, CONTEXT.canonical_refs:100 references it, but the repo's `gateway/docs/` contains 10 runbooks and none named `RUNBOOK-OPS.md`. Recorded as gap #7 for plan 09 to disposition (create new file vs merge into RUNBOOK-PRIMARY-POD.md).
