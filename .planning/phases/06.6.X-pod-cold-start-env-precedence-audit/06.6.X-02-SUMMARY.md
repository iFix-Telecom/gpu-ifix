## Key Findings

1. **No divergence** for the 4 audit-scope shape vars: `stack.env` and container `Config.Env` are in lockstep (`RTX 3090`, `2`, `0.60`, `43803,55158`).
2. **Stack location moved**: PLAN.md assumed `/opt/ai-gateway-dev/`, but that path was renamed to `/opt/ai-gateway-dev.DEPRECATED-portainer-migration-2026-06-05` on 2026-06-05. The live runtime config lives in Portainer at `/data/docker/volumes/infra_portainer_data/_data/compose/34/` — confirmed via `docker inspect ai-gateway-dev` labels (`com.docker.compose.project.working_dir=/data/compose/34/gateway`).
3. **Distroless container caveat**: `docker exec ai-gateway-dev env` returns empty because the image is distroless (no shell, no `env` binary). Truth-of-runtime is `docker inspect --format '{{range .Config.Env}}...'`. PLAN.md's command set would have produced a false-empty result.
4. **Namespace drift flagged for plan 09**: container env contains 3 var families simultaneously: operator-set `PRIMARY_*` (audit scope), compose-default `PRIMARY_VAST_*_PRIMARY/FALLBACK` (separate values), and an unprefixed legacy `VAST_PRICE_CAP_DPH=0.40` (different from `PRIMARY_VAST_PRICE_CAP_DPH=0.60`). Which one the scheduler reads is the precedence question plan 09 must resolve.
5. **Sha256 fingerprints recorded** for both `stack.env` and the deprecated `.env` so any later operator edit between phase plans is detectable.
