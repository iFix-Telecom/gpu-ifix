### Naming-namespace observation (for plan 09)

The 4 operator-set audit vars use the `PRIMARY_*` namespace (no `VAST_` infix for some, e.g. `PRIMARY_GPU_NAME`, `PRIMARY_NUM_GPUS`), but the compose.yml also accepts a parallel `PRIMARY_VAST_*_PRIMARY` / `PRIMARY_VAST_*_FALLBACK` namespace with separate defaults. This is exactly the precedence drift Phase 6.6.X exists to diagnose. Plan 09 should resolve which namespace the code actually reads at scheduler time.
