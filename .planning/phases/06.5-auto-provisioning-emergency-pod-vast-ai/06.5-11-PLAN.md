    ## Incident Playbook

    ### Budget overrun (Sentry alert: budget_exceeded)

    **Symptom:** Sentry alert `subsystem:emerg alert:budget_exceeded` recebido. Email/Slack notification.

    **Diagnosis:**
    1. `gatewayctl emerg lifecycles --since 30d` — quantas lifecycles e qual é o spend total?
    2. Cross-check Vast.ai bill: https://cloud.vast.ai/billing/
    3. Identificar lifecycles caras: `gatewayctl emerg lifecycles --format=json | jq 'sort_by(-.total_cost_brl) | .[:5]'`

    **Action:**
    - Se previsto (UAT/teste): cleanup pre-seed rows
    - Se inesperado: investigate `trigger_reason` distribution; se `failed_over_sustained` dominate, primário Vast.ai precisa investigation (Phase 1 stability)
    - Update env: `MONTHLY_EMERGENCY_BUDGET_BRL=400` se trend é growing legitimately

    ### Zombie pod (Sentry alert: leader_recovery_zombie)

    **Symptom:** Sentry alert `shutdown_reason:leader_recovery_zombie` — nova lider achou instance Vast em estado terminal.

    **Diagnosis:**
    1. `gatewayctl emerg lifecycles --since 1h` — qual lifecycle ID?
    2. `vast list` (via vast-ai.sh) — instance ainda existe?
    3. Verificar audit row events JSONB para timeline

    **Action:**
    - Recovery automático já destruiu o pod (per D-D5); apenas confirmar e arquivar
    - Investigar root cause de leader churn: `kubectl logs ai-gateway-dev-1 | grep "lost leadership"`
    - Se Redis flapping: investigate infra-redis-1 health

    ### Unique violation (DB error: emergency_live_singleton)

    **Symptom:** Logs mostram `ERROR: duplicate key value violates unique constraint "emergency_live_singleton"`.

    **Diagnosis:**
    1. `psql -c "SELECT * FROM ai_gateway.emergency_lifecycles WHERE ended_at IS NULL"` — esperado: 1 row; se 2+ aparece, é split-brain Pitfall
    2. `gatewayctl emerg state --format=json` em cada réplica (deve estar 1 com state != healthy)

    **Action:**
    - Se 2+ live rows: corruption — manualmente close 1 (`UPDATE ... SET ended_at=NOW(), shutdown_reason='manual_split_brain_recovery' WHERE id=...`)
    - Investigate redsync logs (Pitfall 4 quorum case)
    - Considerar restart das 2 réplicas para reset state

    ### Vast API down (vast.Ping fails or transport errors)

    **Symptom:** Logs `vast.Ping falhou` ou `gateway_vast_api_requests_total{status="transport_error"}` spike.

    **Diagnosis:**
    1. `curl https://vast.ai/api/v0/users/current -H "Authorization: Bearer ${VAST_AI_API_KEY}"` direto do host
    2. https://vast.ai/status para outage status
    3. Verificar API key valid: https://cloud.vast.ai/account/
