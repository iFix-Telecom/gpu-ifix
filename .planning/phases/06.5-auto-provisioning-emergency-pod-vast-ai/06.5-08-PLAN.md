<objective>
Fechar o ciclo end-to-end emergencial: implementar **cutback** (5min healthy → routing volta para primário) + **idle destroy** (5min grace → pod terminated) + **multi-failover ride-out** (D-C4 — não cria lifecycles redundantes durante ACTIVE) + integração dispatcher (atomic.Pointer override + restore). Sem este plan, lifecycle entra em ACTIVE mas nunca termina — pod fica indefinido + budget runaway.

Purpose: Este é o plan que prova **SC-4** ("primário healthy 5min → cutback; +5min idle → destroy"). Também implementa D-C4 (ride-out) que previne o pior cenário de cost runaway: oscilação chatty disparando 5+ lifecycles em poucos minutos.

Output: 2 modificações em packages externos (upstreams/loader, proxy/dispatcher) + 2 modificações em emerg + 3 integration tests cobrindo cutback, idle destroy, ride-out.
</objective>
