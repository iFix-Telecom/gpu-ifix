<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → authClient (change/reset password) | operator-scoped credential change; built-in Better Auth verification |
| reset link → /reset-password/[token] | token from email crosses an unauthenticated surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-selfpw | Spoofing | change-password without current pw | mitigate | authClient.changePassword requires currentPassword; Better Auth verifies server-side; wrong → error code |
| T-13-token | Tampering | reset token replay / wrong origin | mitigate | token consumed by Better Auth resetPassword (single-use verification table); link origin = BETTER_AUTH_URL public (Pitfall 6, confirmed Plan 02) |
| T-13-disclosure | Information Disclosure | rendering token/credential | mitigate | privacy rule — token never shown; only password inputs; no audit of self-pw (D-09) |
</threat_model>
