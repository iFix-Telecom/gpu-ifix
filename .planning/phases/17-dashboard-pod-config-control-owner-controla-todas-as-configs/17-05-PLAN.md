<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → server action | requireOwner runs server-side; UI hiding is cosmetic only. |
| server action → gateway admin API | X-Admin-Key held server-only in gateway-admin.ts; never serialized to the client. |
| dashboard → ai_gateway schema | Dashboard writes ONLY via the gateway endpoint (never direct DB); audit goes to dashboard admin_audit_log (D-06). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-15 | Elevation of Privilege | operator edits config via the action | mitigate | requireOwner FIRST, server-side (not UI-hiding); operator throws + no gateway call + no audit (Task 1 behavior 2) |
| T-17-16 | Information Disclosure | admin key leaks to the browser | mitigate | Key read only in server-only gateway-admin.ts + route.ts; client wrappers key-free; key-leak grep enforces (Task 2) |
| T-17-17 | Tampering | malicious value bricks prod | mitigate | Server-side bounds validation BEFORE the gateway call (defense-in-depth with the gateway's own check, Plan 17-04) |
| T-17-18 | Repudiation | config change with no trail | mitigate | Exactly one admin_audit_log row per edit (who/when/field/old/new), action="pod_config.update"/"pod_config_bounds.update" (D-06) |
| T-17-SC | Tampering | npm install (no new deps expected) | accept | No new packages; reuses Phase 13/15 deps. If a form lib is proposed, run the Package Legitimacy Gate (none expected — RESEARCH N/A) |
</threat_model>
