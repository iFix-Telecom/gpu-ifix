# Phase 7 — UI Design Contract

> Visual and interaction contract for the new `dashboard/` Next.js 15 app. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** an internal operational dashboard for ~4 Ifix operators/admins. It surfaces live per-tenant latency (P50/P95/P99), error rate, daily/monthly cost per tenant, the current failover FSM state, an incident history / audit-log view, and a critical-event banner. This is a monitoring screen — information density, scannability, and at-a-glance status outrank marketing polish.

## Visual Hierarchy

**Primary focal point:** the critical-event banner (sticky top, full-width, `--destructive` red) when an active critical alert exists; otherwise the KPI display values (28px semibold, top KPI row) are the dominant element the operator's eye lands on first.

Secondary tier: the FSM-state panel and per-tenant metrics table. Tertiary tier: charts and the audit-log panel. The single accent-colored primary action button per view is the only deliberate accent draw within an otherwise neutral/dark surface.

---

## Spacing Scale

The spacing scale contains **only** the standard token set `{4, 8, 16, 24, 32, 48, 64}` — every margin, gap, and padding value in this phase resolves to one of these seven tokens. There are **no exceptions** to the scale. Fixed component box heights that are not members of this set are deliberately kept out of the scale and documented separately under [Layout Constraints](#layout-constraints) below.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, metric-value sub-labels, badge inset |
| sm | 8px | Compact spacing inside dense metric cards and table cells |
| md | 16px | Default gap between cards, form fields, card inner padding |
| lg | 24px | Card group padding, section header margin |
| xl | 32px | Gap between dashboard sections (KPI row → charts → audit table) |
| 2xl | 48px | Page top padding below the global banner/header |
| 3xl | 64px | Reserved — not expected on a density-first ops screen |

**Exceptions: none.** The spacing scale is exactly `{4, 8, 16, 24, 32, 48, 64}` with nothing added or substituted.

## Layout Constraints

This section is **not part of the spacing scale.** The values below are fixed pixel heights of two specific component boxes — they size a component, they do not describe space between or inside elements. They are listed here, in their own top-level section outside the Spacing Scale, precisely so they are never read as spacing-scale tokens. The internal padding of both components still uses the spacing scale above (`sm` = 8px).

| Component | Fixed height | Type | Justification |
|-----------|--------------|------|---------------|
| Data table row | 36px | Component height constraint — not a spacing token | Deliberate compact-table row height so the audit-log and per-tenant tables stay scannable. Composed of `sm` (8px) vertical padding × 2 + a 20px text line. A 32px row would force padding below `sm` and break cell legibility on a dense ops screen; a 48px row wastes vertical space and reduces the number of visible rows. |
| Critical-event banner | 44px min height | Component height constraint — not a spacing token | Fixed sticky-bar minimum height — internal padding uses `sm` (8px), but the banner box is pinned to a 44px floor so the persistent status target stays a consistent size regardless of copy length, and meets the conventional 44px minimum pointer/touch target. |

## Typography

shadcn radix-nova default sans (`--font-sans`). Exactly 4 sizes, exactly 2 weights.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 — table cells, audit-log entries, descriptive text, default app body (ops screens run denser than the 16px marketing default) |
| Label | 12px | 600 (semibold) | 1.4 — KPI card captions, table column headers, badge text, axis labels |
| Heading | 20px | 600 (semibold) | 1.2 — section headers ("Tenants", "Failover State", "Incident History") |
| Display | 28px | 600 (semibold) | 1.2 — primary KPI values (P95 ms, error rate %, monthly cost BRL) — the at-a-glance numbers |

Tabular numerals (`font-variant-numeric: tabular-nums`) required on all Display and Body cells that render metrics (latency, cost, counts) so digits align in tables and don't jitter on 5–10s React Query refetch.

## Color

radix-nova `.dark` OKLCH tokens, verified in converseai-v4 `globals.css`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `oklch(0.13 0.028 261.692)` — `--background` | App background, page surface |
| Secondary (30%) | `oklch(0.21 0.034 264.665)` — `--card` / `--sidebar` | KPI cards, charts panels, sidebar nav, audit table container |
| Accent (10%) | `oklch(0.648 0.2 131.684)` — `--primary` (green) | See reserved-for list below |
| Destructive | `oklch(0.704 0.191 22.216)` — `--destructive` (red) | Critical-event banner background, critical alert badges, "GPU down" / `FAILED_OVER` FSM state, error-rate KPI when over threshold |

Accent reserved for:
- The single primary action button per view (e.g. **Reconhecer incidente** on the banner, **Aplicar período** on a date-range filter)
- The active sidebar nav item
- Focus rings on interactive controls
- The "healthy" / `HEALTHY` FSM state indicator and the in-SLO latency state

Accent is **not** used for: every chart series, every link, table row hover, or generic borders.

### Semantic status palette (ops-specific — Claude's Discretion)

The converseai-v4 chart palette (`--chart-1..5`) is five monochrome green steps — insufficient to distinguish *status* on a monitoring screen. This phase adds a 3-tier status palette mapped onto existing radix-nova tokens (no new base colors invented):

| Status | Token | OKLCH | Applied to |
|--------|-------|-------|------------|
| Healthy / in-SLO | `--primary` (green) | `oklch(0.648 0.2 131.684)` | `HEALTHY` FSM badge, P50 line, error rate under threshold, cost under quota |
| Warning / degraded | `--chart-1` amber-substitute → declare `--status-warning: oklch(0.769 0.188 70.08)` | `oklch(0.769 0.188 70.08)` (amber) | `DEGRADED` / `RECOVERING` / `COOLDOWN` / `EMERGENCY_PROVISIONING` FSM badges, P95 line, error rate 1–5%, cost at 75–90% of quota, warning-tier alerts |
| Critical / failed | `--destructive` (red) | `oklch(0.704 0.191 22.216)` | `FAILED_OVER` / `EMERGENCY_ACTIVE` FSM badges, P99 line, error rate over 5%, cost over 90% of quota, critical banner |
| Neutral / off | `--muted-foreground` | `oklch(0.707 0.022 261.325)` | `OFF_HOURS` / `MAINTENANCE` FSM badges, disabled rows, "no data" placeholders |

`--status-warning` is the only token added beyond radix-nova; it is declared once in `globals.css` alongside the imported preset variables. Chart series for P50/P95/P99 use green / amber / red respectively so the three percentile lines are distinguishable at a glance.

## Copywriting Contract

Audience: ~4 internal Ifix operators. Copy is **operational and direct** — no marketing tone, no exclamation. Portuguese (pt-BR) to match the operators' working language and the existing Ifix runbooks/CLAUDE.md conventions.

| Element | Copy |
|---------|------|
| Primary CTA | **Reconhecer incidente** — on the critical-event banner; acknowledges the active critical alert and collapses the banner (does not resolve the underlying condition) |
| Date-range confirm button | **Aplicar período** — confirms the selected `from`/`to` range on the cost/usage date-range filter (the component-inventory `button` block; pt-BR, not "Apply") |
| Empty state heading | **Sem dados no período** |
| Empty state body | Nenhuma requisição registrada para este tenant no intervalo selecionado. Ajuste o período ou confirme que o tenant está roteando pelo gateway. |
| Error state | Não foi possível carregar as métricas do gateway. Verifique se o gateway está no ar e se a admin-key está válida, depois use **Tentar novamente**. |
| Destructive confirmation | Não há ação destrutiva no Phase 7. O dashboard é **somente leitura** — lê `/admin/metrics` e `audit_log` via admin-key; não escreve estado no gateway. "Reconhecer incidente" só afeta a UI local (silencia o banner por 5 min, espelhando o TTL de dedupe do alerting) e não dispara confirmação. |

Additional fixed strings (for consistency across cards and tables):

| Context | Copy |
|---------|------|
| Loading state | Carregando métricas… |
| Stale-data indicator | Atualizado há {n}s — appears next to the page title when the last successful React Query refetch is older than the 5–10s poll interval |
| Banner — critical active | GPU primária indisponível há {duração}. Failover ativo. — red banner, sticky top |
| Banner — warning active | Saturação sustentada / taxa de erro elevada — {detalhe}. — amber banner |
| FSM state labels | `HEALTHY` → Saudável · `DEGRADED` → Degradado · `FAILED_OVER` → Em failover · `EMERGENCY_PROVISIONING` → Provisionando pod · `EMERGENCY_ACTIVE` → Pod emergencial ativo · `RECOVERING` → Recuperando · `COOLDOWN` → Cooldown · `OFF_HOURS` → Fora de horário · `MAINTENANCE` → Manutenção |
| Audit-log empty | Nenhum evento registrado no período. |
