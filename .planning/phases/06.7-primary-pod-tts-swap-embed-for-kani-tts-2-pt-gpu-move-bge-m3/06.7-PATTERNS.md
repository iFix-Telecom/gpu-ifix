### `gateway/internal/proxy/tts.go` (NEW — proxy, JSON→binary)

**Analog estrutural:** `gateway/internal/proxy/audio.go` (constructor shape) — **NÃO copiar a semântica multipart**. A direção de dados é oposta: STT é multipart→JSON, TTS é JSON→binary audio. Reusar apenas `BuildDirector` + `ErrorHandler` + `ComposeInterceptors`.

**Constructor pattern a espelhar** (`audio.go:19-40`):
```go
func NewAudioProxy(upstreamURL string, log *slog.Logger, interceptors ...ProxyResponseInterceptor) (*httputil.ReverseProxy, error) {
	u, err := url.Parse(upstreamURL)
	if err != nil { return nil, fmt.Errorf("proxy/audio: parse %q: %w", upstreamURL, err) }
	if u.Scheme == "" || u.Host == "" { return nil, fmt.Errorf("proxy/audio: invalid upstream url %q", upstreamURL) }
	rp := &httputil.ReverseProxy{
		Director: BuildDirector(u),
		Transport: &http.Transport{
			MaxIdleConns:          20,
			MaxIdleConnsPerHost:   4,
			IdleConnTimeout:       90 * time.Second,
			ResponseHeaderTimeout: 60 * time.Second,  // TTS: subir p/ síntese longa (RESEARCH §Pattern 3 sugere 30-60s)
		},
		ErrorHandler:   ErrorHandler("stt", log),       // TTS: ErrorHandler("tts", log)
		ModifyResponse: ComposeInterceptors(interceptors...),
	}
	return rp, nil
}
```

**Diretiva crítica (RESEARCH Pattern 3 + Anti-Pattern):**
- `FlushInterval` default 0 (buffered) p/ non-streaming — igual `audio.go`/`embeddings.go`. SE suportar `stream_format:sse`, usar `FlushInterval=-1` (ver `director.go:1-7` doc do chat streaming).
- `ErrorHandler("tts", log)`.
- `Director` = `BuildDirector(u)` — preserva path 1:1 e strip de auth headers do cliente (ver excerpt do director abaixo).

**Director compartilhado** (`director.go:39-62`) — usar como-está, NÃO reescrever:
```go
func BuildDirector(upstream *url.URL) func(*http.Request) {
	return func(r *http.Request) {
		r.URL.Scheme = upstream.Scheme
		r.URL.Host = upstream.Host
		r.Host = upstream.Host
		for _, h := range clientAuthHeaders { r.Header.Del(h) } // Authorization, X-API-Key, Cookie, Proxy-Authorization
		if rid := httpx.RequestIDFrom(r.Context()); rid != "" {
			r.Header.Set("X-Request-ID", rid)
		} else { r.Header.Del("X-Request-ID") }
	}
}
```

**ErrorHandler envelope** (`errors.go:35-48`) — copiar assinatura, trocar nome:
```go
func ErrorHandler(upstreamName string, log *slog.Logger) func(http.ResponseWriter, *http.Request, error) {
	log = log.With("module", "PROXY", "upstream", upstreamName)
	return func(w http.ResponseWriter, r *http.Request, err error) {
		log.ErrorContext(r.Context(), "upstream error", "err", err, "request_id", httpx.RequestIDFrom(r.Context()), "path", r.URL.Path, ...)
		httpx.WriteOpenAIError(w, http.StatusBadGateway, "api_error", "upstream_unreachable", "The upstream inference service is temporarily unreachable.")
	}
}
```

### `gateway/internal/proxy/voices.go` (NEW — CRUD handler, S3 file-I/O + Postgres)

**NÃO existe cliente S3 em Go no repo hoje** — MinIO é usado apenas pod-side via env-forwarding (`lifecycle.go:322-325`) consumido por `mc` no onstart. Este handler é o **primeiro** cliente S3 Go. Mapear de 3 analogs:

**1. Tenant isolation (D-10)** — toda query filtra por `TenantID` do contexto, NUNCA do body (`auth/context.go:13-39`):
```go
type AuthContext struct {
	TenantID  string
	APIKeyID  string
	DataClass DataClass
	KeyPrefix string
}
func MustFromContext(ctx context.Context) AuthContext { /* panics if middleware not applied */ }
```
Uso no handler: `ac := auth.MustFromContext(r.Context()); tenant := ac.TenantID` → passar como `WHERE tenant_id = $tenant` (Security Domain V4).

**2. Credenciais MinIO** — já em `config.Config` (`config.go:199-202`): `MinioEndpoint`, `MinioBucket`, `MinioAccessKey`, `MinioSecretKey`. O handler recebe um cliente S3 construído no boot (wiring em `main.go` `px` struct, ver abaixo). Library: usar `github.com/aws/aws-sdk-go-v2` (CLAUDE.md confirma `@aws-sdk/client-s3` como padrão S3 do ecossistema — usar o equivalente Go) ou MinIO Go SDK; decisão do planner. Chave S3 derivada de UUID, NÃO do input (Security V12 / threat "Path traversal via voice_id").

**3. Body cap (DoS)** — `http.MaxBytesHandler` já aplicado no router (`main.go:985` per RESEARCH); aplicar à rota `/v1/audio/voices`. Regex allowlist do `voice_id` espelha o padrão Piper `^[A-Za-z0-9_\-\.]+$` (RESEARCH Security Domain).

**4. Handler shape** — seguir o estilo dos handlers proxy (constructor recebe deps, retorna `http.Handler`), mas como é CRUD nativo (não reverse-proxy), o corpo é manual: parse JSON/multipart → sqlc query → S3 PutObject/GetObject → `httpx.WriteOpenAIError` em erro.
