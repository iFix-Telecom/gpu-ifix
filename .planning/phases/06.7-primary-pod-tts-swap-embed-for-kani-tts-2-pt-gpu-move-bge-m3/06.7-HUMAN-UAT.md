## Scenario S2 — Zero-shot voice-clone round-trip (24kHz cloned voice)

**Goal:** Upload reference WAV → `voice_id` → `/v1/audio/speech` returns 24kHz audio in the cloned voice.

```bash
# upload reference WAV
curl -sS -H "Authorization: Bearer $KEY" \
  -F "file=@reference.wav" -F "label=uat-voice" \
  $GW/v1/audio/voices
# -> {"voice_id":"<uuid>","label":"uat-voice"}   (export VID=<uuid>)

# synth in that voice
```

**Expected:** `voice_id` returned; `s2_cloned.wav` is a 24000 Hz WAV; the voice audibly matches the reference (operator listens). MinIO has `voices/$VID.wav`.

## Scenario S3 — VOICE SURVIVES POD REPLACEMENT (S3 WAV refetch, NO `.pt`) [REVIEWS #1]

## Execution notes (2026-05-21)

**Bugs found + fixed live (all committed to `develop`):**
1. CI: sqlc gen out of sync; gofmt unclean main.go
2. Piper adapter sent form-encoded; voice-api Piper expects JSON `/tts`
3. torch cu124 → **cu128** (RTX 5090 Blackwell sm_120 — no kernel image)
4. **CUPTI**: cu128 torch needs nvidia-cuda-*-cu12 12.8 runtime (dropped `--no-deps`)
5. **setuptools<81**: resemble-perth uses `pkg_resources` (removed in setuptools 81+) → PerthImplicitWatermarker None
6. audit middleware truncated the multipart voice upload (isAudioRoute only matched /transcriptions)
7. **TTS routing**: loader override resolves name `emergency_pod_tts`, but dispatcher had no proxy registered → 503; added `NewDynamicTTSProxy`
8. Host selection: removed `verified` + EU-`geolocation` filters; lowered `PRIMARY_VAST_PRICE_CAP_DPH` 2.20→1.00; added `PRIMARY_VAST_MACHINE_BLOCKLIST` + machine_id logging (catalog broken-CDI hosts: 36773, 53128, 38389)

**Known latent gap (out of scope):** llm/stt have the same `emergency_pod_<role>` no-registered-proxy issue as TTS had — they route to a dummy `UPSTREAM_*_URL` and were only ever exercised pod-direct, not through the gateway dispatcher.

**Redesign decided (future planned phase — see seed):** emergency pod becomes a FULL primary-pod hot-standby on a 5090 (LLM+STT+TTS, same image/weights/onstart/GPU/cap/blocklist/filter as primary), and embed returns to the pod as tier-0 (GPU) with the 24/7 CPU Infinity as tier-1 — reversing 06.7 D-03. Target topology: every role tier-0=pod-GPU, tier-1=always-on fallback (LLM→OpenRouter, STT→OpenAI Whisper, TTS→Piper, embed→CPU 24/7).
