# Phase 06.7 Plan 08: Primary Reconciler TTS Swap + Pitfall #11 Re-assert Summary

The primary reconciler now drives the `tts` tier-0 slot (Chatterbox on pod port 8003) instead of `embed`, and re-asserts any cleared `llm`/`stt`/`tts` slot on each 1Hz Ready-tick — closing the emerg-cutback-clears-the-primary-slot bug (tech debt #9 / Pitfall #11) across all three dynamic roles. The pod lifecycle's port wiring flipped from embed:8002 to tts:8003.

## What Was Built

### Task 1 — reconciler: embed→tts swap + Pitfall #11 Ready-tick re-assert (commit 53f0e2f)
- `markReady` and `recoverOpenLifecycle` now `OverrideTier0("llm"/"stt"/"tts")` — `embed` removed entirely (D-03; embed is a static off-pod tier-0 row).
- `startDrain` and `closeLifecycle` `RestoreTier0("llm"/"stt"/"tts")`.
- `evaluateReady` gained the Pitfall #11 / D-13 fix: when `activePodURLs != nil` and a Loader is wired, it loops `{llm,stt,tts}` and, for any role whose `Tier0OverrideURL(role)` returns `set=false`, re-calls `OverrideTier0(role, stripPrimaryReadinessSuffix(roleURL(*urls, role)))` plus a `Warn` log. `embed` is excluded. The loop runs even under `PRIMARY_POD_SCHEDULE_DISABLED`.
- `LoaderAdapter` interface gained `Tier0OverrideURL(role string) (string, bool)`.
- 4-endpoint health-check (forward poll + restart recovery) now probes `urls.TTS` instead of `urls.Embed`.
- The two Plan 02 owned stubs (`TestEvaluateReady_ReassertsTier0WhenCleared`, `TestMarkReady_OverridesTTSNotEmbed`) are unskipped and assert real behavior (re-assert fires for llm/stt/tts, never embed; markReady overrides tts not embed).

### Task 2 — lifecycle: port 8002→8003, podTTSURL, struct + port map (commit facdec0)
- `primaryPodURLs.Embed` field renamed to `TTS`.
- `podEmbedURL` (8002 /health) replaced by `podTTSURL` (8003 /health); `buildPodURLs` sets `TTS`.
- Port-forward map drops `-p 8002:8002`, adds `-p 8003:8003` (Chatterbox child).
- MINIO_* creds remain forwarded (pod's lazy voice-WAV fetch — Plan 05 zero-shot contract).
- Added `roleURL(urls primaryPodURLs, role string) string` helper used by the re-assert loop.

The plan's `files_modified` lists only `reconciler.go` and `lifecycle.go`, but the behavior/interface changes directly broke sibling test files that assert the old `embed`/`8002` contract or implement the `LoaderAdapter` interface. ### Env-map: BGE-M3 weights forward RETAINED (Rule 3 — avoid breaking out-of-scope onstart.go)

The Task 2 action says "remove the embed weights forward (`PRIMARY_BGEM3_WEIGHTS_KEY`/`_SHA256`)". ### Chatterbox weights/model env: NOT forwarded (follows Plan 05 as-built)

The Task 2 action says "add the Chatterbox weights/model key/sha forward in the identical format". Per Plan 05 SUMMARY + the as-built `pod/primary/supervisord.conf`, Chatterbox downloads its weights from HuggingFace at pod onstart (not MinIO) and the `CHATTERBOX_*` config is baked into supervisord with defaults. There is no MinIO weights key/sha to mirror. Only the MINIO_* creds (already forwarded, retained) are needed for the lazy voice-WAV fetch. No Chatterbox env was added to the lifecycle env map.
