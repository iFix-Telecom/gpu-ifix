### [Rule 1 - Bug] audit_log.upstream uniformly recorded route-derived value

**Found during:** Task 2 cenário 2 first run.
**Issue:** audit_log.upstream wrote `'stt'` for every STT request — D-B13
acceptance criteria (per-scenario `audit_log.upstream='gemini-stt'`/`'groq-whisper'`/etc.)
was machine-unverifiable. Pre-existing bug surfaced by Phase 11.2 cascade
(in Phase 11.1 single-fallback there was no distinction need).

**Fix attempts:**
1. **Attempt 1 (commit 145be87):** Mutate request struct in dispatcher.dispatchTo
   via `*r = *r.WithContext(WithUpstreamOverride(name))`. Failed: tests showed
   `TestChatProxy_SSEStreamingFlushesPerChunk` flaky failure, but root cause
   was different — `http.TimeoutHandler` clones the request before passing
   to inner handler, so audit middleware (sitting OUTSIDE TimeoutHandler)
   captured the pre-clone *Request reference; in-handler mutation invisible.
2. **Attempt 2 (commit 58590fc) — LANDED:** Request-id-keyed sync.Map
   registry in auditctx. Producer `SetDispatchedUpstream(reqID, upstream)`
   from `dispatcher.dispatchTo` + `dispatchOverride`. Consumer
   `TakeDispatchedUpstream(reqID)` from audit middleware (take-on-read
   keeps registry bounded). Bypasses ctx propagation entirely.

**Files modified:** gateway/internal/proxy/dispatcher.go,
gateway/internal/audit/middleware.go, gateway/internal/auditctx/override.go
**Verification:** Cenários 2-4 audit rows now show distinct upstreams
(`gemini-stt`, `groq-whisper`, `openai-whisper`) post-deploy.
**Tests:** All gateway test suites GREEN locally + GHA (build-gateway

### [Rule 1 - Bug] uat-11.2.sh audit query missed when upstream clobbers X-Request-ID

**Found during:** Task 2 cenário 3 second run (after Groq URL fix).
**Issue:** When dispatch target is OpenAI-compat external (Groq),
`httputil.ReverseProxy` preserves the upstream's `X-Request-ID` response
header, clobbering the gateway UUID. Script's `WHERE request_id = ...`
lookup returned empty because captured header was Groq's `req_*` id.

**Fix (commit 9e36ca4):**
- `audit_query_upstream` skips request_id lookup when value doesn't match
  UUIDv7 regex (`^[0-9a-f]{8}-[0-9a-f]{4}`)
- Fallback: `SELECT … WHERE route='/v1/audio/transcriptions' AND ts > NOW() - INTERVAL '30 seconds'`
- Single-flight UAT execution makes the most-recent row the authoritative
  match for the previous POST
- Same pattern applied to `audit_query_data_class` for cenário 5

**Files modified:** pod/smoke/uat-11.2.sh
**Verification:** Cenário 3 third run PASS with audit upstream correctly
read via fallback.
