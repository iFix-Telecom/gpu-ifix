**Multipart parsing pattern** (analog lines 174-268 `rewriteMultipartModelViaResolver`):
- `mime.ParseMediaType(contentType)` → boundary extraction
- `multipart.NewReader(bytes.NewReader(body), boundary)` iteration loop
- `part.FormName() == "file"` branch reads audio bytes via `io.ReadAll(part)`
- Non-file parts drained via `io.Copy(io.Discard, part)`

Gemini director extracts the `file` part bytes only (NOT model rewrite) and base64-encodes for inline_data — see RESEARCH §Pattern 2 `extractAudioFromMultipart` (lines 386-419).

**Breaker IsSuccessful contract** — analog comments (lines 21-23): `breaker's IsSuccessful filter classifies 4xx as non-failure so the breaker stays CLOSED`. Same applies for Gemini 4xx (413 oversize, malformed multipart).

```go
func (l *Loader) ResolveAllTier1(role string) []UpstreamConfig {
    s := l.snap.Load()
    if s == nil { return nil }
    var out []UpstreamConfig
    for _, u := range s.ordered {
        if u.Role == role && u.Tier == 1 && u.Enabled {
            out = append(out, u)
        }
    }
    sort.SliceStable(out, func(i, j int) bool {
        return out[i].TierPriority < out[j].TierPriority
    })
    return out
}
```

### Breaker IsSuccessful Contract

**Source:** Pattern documented in `openai_whisper_director.go` lines 21-23.

**Apply to:** All STT directors. 4xx responses (413 oversize, 415 unsupported MIME, 400 malformed multipart) are classified as non-failure — breaker stays CLOSED. Only 5xx + transport errors trip the breaker.
