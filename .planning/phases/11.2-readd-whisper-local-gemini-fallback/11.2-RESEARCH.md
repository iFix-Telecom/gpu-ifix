# Phase 11.2 — Re-add Whisper Local + Gemini 2.5 Flash Lite Fallback — Research

**Researched:** 2026-06-05
**Domain:** Multi-provider STT cascade (tier-0 local Whisper → tier-1 Gemini → tier-1 OpenAI safety net) + Gemini 2.5 Flash Lite audio API integration
**Confidence:** HIGH (revert paths verified via git history; Gemini API CITED from official docs; multi-tier-1 schema gap surfaced via dispatcher code read)

## Summary

Phase 11.2 reverts the Phase 11.1 shrink (Whisper/Speaches removal) AND adds a NEW
Gemini 2.5 Flash Lite tier-1 primary fallback in front of the existing OpenAI Whisper
safety net. Three providers cascade for `/v1/audio/transcriptions`:

1. **local-stt** (tier-0, pod-resident Speaches/faster-whisper-large-v3) — $0/h when pod ON
2. **gemini-stt** (tier-1 primary fallback) — ~$0.30/M audio tokens ≈ $0.05/h batch
3. **openai-whisper** (tier-1 safety net) — $0.36/h (kept for Gemini quota/breaker overflow)

**Primary recommendation:** Revert Phase 11.1 plans 01/02/03/04 with surgical precision
(git history provides exact pre-state), THEN add **one new tier-1 upstream slot** for
Gemini via a schema change (a `tier_priority` column added to `ai_gateway.upstreams`)
because the current `UNIQUE (role, tier)` constraint allows only ONE tier-1 per role.
Gemini audio API is straightforward (`x-goog-api-key` + `inline_data` base64 for
<20 MB files, response in `candidates[0].content.parts[0].text`). The biggest unknown
is the multi-tier-1 dispatcher fall-through behavior — covered in Pitfall 1.

### Pattern 2: Gemini STT director adapter (NEW)

**What:** Translate OpenAI-shaped multipart `POST /v1/audio/transcriptions` → Gemini `POST /v1beta/models/<model>:generateContent` with `inline_data`, flatten response back to `{"text": "..."}`.

**When to use:** New tier-1 upstream where the wire shape differs from OpenAI's spec.

**Example:**

```go
// Source: synthesised from openai_whisper_director.go (existing pattern) + ai.google.dev/gemini-api/docs/audio
// File: gateway/internal/proxy/gemini_stt_director.go

package proxy

import (
    "bytes"
    "encoding/base64"
    "encoding/json"
    "fmt"
    "io"
    "log/slog"
    "mime"
    "mime/multipart"
    "net/http"
    "net/http/httputil"
    "net/url"
    "strings"

    "github.com/ifixtelecom/gpu-ifix/gateway/internal/models"
)

// geminiMaxInlineBytes is the cutoff for inline_data base64 (~20 MB request body).
// Over this we'd need the resumable files.upload protocol — deferred to a future
// phase per CONTEXT D-B4 (sync path only for v1).
const geminiMaxInlineBytes = 20 * 1024 * 1024

const geminiTranscribePrompt = "Transcribe this audio. Return only the transcription text, no commentary."

type geminiInlineData struct {
    MimeType string `json:"mime_type"`
    Data     string `json:"data"` // base64
}

type geminiPart struct {
    Text       string            `json:"text,omitempty"`
    InlineData *geminiInlineData `json:"inline_data,omitempty"`
}

type geminiContent struct {
    Parts []geminiPart `json:"parts"`
}

type geminiRequest struct {
    Contents []geminiContent `json:"contents"`
}

type geminiResponse struct {
    Candidates []struct {
        Content struct {
            Parts []struct {
                Text string `json:"text"`
            } `json:"parts"`
        } `json:"content"`
    } `json:"candidates"`
    Error *struct {
        Code    int    `json:"code"`
        Message string `json:"message"`
        Status  string `json:"status"`
    } `json:"error,omitempty"`
}

// BuildGeminiSTTDirector returns a *httputil.ReverseProxy-compatible Director +
// ModifyResponse pair. Unlike the OpenAI Whisper director (which only rewrites
// the multipart "model" field), Gemini requires BODY + URL + METHOD + HEADER
// rewrites because the wire shape is entirely different (JSON, not multipart).
//
// Director rewrites:
//   - URL path → /v1beta/models/<model>:generateContent
//   - Header Authorization → x-goog-api-key: <bearer>  (Gemini uses API key, NOT OAuth)
//   - Body multipart → JSON { contents: [{ parts: [text, inline_data] }] }
//   - Content-Type → application/json
//
// ModifyResponse:
//   - Read Gemini JSON, extract candidates[0].content.parts[0].text
//   - Rewrite body to { "text": "<extracted>" } (OpenAI Whisper response shape)
//   - On Gemini error envelope → translate to OpenAI error envelope (502)
//
// Audio bytes are read into memory once (capped at geminiMaxInlineBytes). Files
// over the cap → respond 413 directly via a sentinel hijack in the director
// (the simplest pattern: stash a flag on r.Context() and have a wrapper handler
// catch it).
func BuildGeminiSTTDirector(
    upstream *url.URL,
    apiKey string,
    resolver *models.Resolver,
    upstreamName string,
    log *slog.Logger,
) (func(*http.Request), func(*http.Response) error) {
    base := BuildDirector(upstream)
    director := func(r *http.Request) {
        base(r)
        // Resolve target model (env override wins via resolver — D-06).
        model := resolver.Resolve("whisper", upstreamName)
        if model == "" {
            model = "gemini-2.5-flash-lite"
        }
        // Auth: REPLACE Authorization (set by BuildDirector via base) with x-goog-api-key.
        r.Header.Del("Authorization")
        if apiKey != "" {
            r.Header.Set("x-goog-api-key", apiKey)
        }
        ct := r.Header.Get("Content-Type")
        if !strings.HasPrefix(ct, "multipart/form-data") || r.Body == nil {
            // Defensive: shouldn't happen on /v1/audio/transcriptions.
            return
        }
        body, err := io.ReadAll(r.Body)
        _ = r.Body.Close()
        if err != nil {
            return
        }
        audioBytes, mimeType, perr := extractAudioFromMultipart(body, ct)
        if perr != nil || len(audioBytes) == 0 {
            // Pass through — upstream will reject; breaker stays CLOSED on 4xx per IsSuccessful.
            r.Body = io.NopCloser(bytes.NewReader(body))
            r.ContentLength = int64(len(body))
            return
        }
        if len(audioBytes) > geminiMaxInlineBytes {
            // Pitfall: oversized payload. Set a header sentinel so an outer
            // wrapper can 413 the client. Simpler: forward an empty JSON body
            // and let Gemini 400; breaker stays CLOSED on 4xx.
            // Plan 04 should decide; recommend explicit 415/413 via wrapper.
            log.Warn("gemini_stt audio exceeds inline cutoff",
                "bytes", len(audioBytes), "cutoff", geminiMaxInlineBytes)
        }
        req := geminiRequest{
            Contents: []geminiContent{{Parts: []geminiPart{
                {Text: geminiTranscribePrompt},
                {InlineData: &geminiInlineData{
                    MimeType: mimeType,
                    Data:     base64.StdEncoding.EncodeToString(audioBytes),
                }},
            }}},
        }
        jsonBody, jerr := json.Marshal(req)
        if jerr != nil {
            return
        }
        r.URL.Path = fmt.Sprintf("/v1beta/models/%s:generateContent", model)
        r.Body = io.NopCloser(bytes.NewReader(jsonBody))
        r.ContentLength = int64(len(jsonBody))
        r.Header.Set("Content-Type", "application/json")
        r.Method = http.MethodPost
    }
    modifyResponse := func(resp *http.Response) error {
        if resp.StatusCode != http.StatusOK {
            return nil // pass through Gemini's error; dispatcher 502 envelope will wrap
        }
        body, err := io.ReadAll(resp.Body)
        _ = resp.Body.Close()
        if err != nil {
            return err
        }
        var gresp geminiResponse
        if jerr := json.Unmarshal(body, &gresp); jerr != nil {
            return jerr
        }
        if gresp.Error != nil {
            // Translate to OpenAI envelope.
            oa, _ := json.Marshal(map[string]any{
                "error": map[string]any{
                    "message": gresp.Error.Message,
                    "type":    "upstream_error",
                    "code":    gresp.Error.Status,
                },
            })
            resp.Body = io.NopCloser(bytes.NewReader(oa))
            resp.ContentLength = int64(len(oa))
            resp.StatusCode = http.StatusBadGateway
            resp.Header.Set("Content-Type", "application/json")
            return nil
        }
        var text string
        if len(gresp.Candidates) > 0 && len(gresp.Candidates[0].Content.Parts) > 0 {
            text = gresp.Candidates[0].Content.Parts[0].Text
        }
        out, _ := json.Marshal(map[string]string{"text": text})
        resp.Body = io.NopCloser(bytes.NewReader(out))
        resp.ContentLength = int64(len(out))
        resp.Header.Set("Content-Type", "application/json")
        resp.Header.Set("Content-Length", fmt.Sprintf("%d", len(out)))
        return nil
    }
    return director, modifyResponse
}

// extractAudioFromMultipart parses the OpenAI-style multipart body and returns
// the file part's bytes + its detected MIME type. Returns the form-field name
// of the file (typically "file"). Drains other parts.
func extractAudioFromMultipart(body []byte, contentType string) ([]byte, string, error) {
    mediaType, params, err := mime.ParseMediaType(contentType)
    if err != nil {
        return nil, "", err
    }
    if !strings.HasPrefix(mediaType, "multipart/") {
        return nil, "", fmt.Errorf("not multipart")
    }
    mr := multipart.NewReader(bytes.NewReader(body), params["boundary"])
    for {
        part, perr := mr.NextPart()
        if perr == io.EOF {
            break
        }
        if perr != nil {
            return nil, "", perr
        }
        if part.FormName() == "file" {
            audio, rerr := io.ReadAll(part)
            _ = part.Close()
            if rerr != nil {
                return nil, "", rerr
            }
            mimeType := part.Header.Get("Content-Type")
            if mimeType == "" {
                mimeType = "audio/wav"
            }
            return audio, mimeType, nil
        }
        _, _ = io.Copy(io.Discard, part)
        _ = part.Close()
    }
    return nil, "", fmt.Errorf("no file part")
}
```

### Pattern 3: Multi-tier-1 fallthrough in dispatcher (NEW)

**What:** When tier-0 is OPEN, iterate tier-1 candidates in `tier_priority` ASC order until the first CLOSED breaker is found. Today's dispatcher (`gateway/internal/proxy/dispatcher.go:257-272`) hardcodes a SINGLE tier-1 lookup.

**Schema change:** add `tier_priority INT NOT NULL DEFAULT 0` to `ai_gateway.upstreams`. The `UNIQUE (role, tier)` constraint must be replaced with `UNIQUE (role, tier, tier_priority)`. Tier-0 rows always carry `tier_priority=0`. Tier-1 rows order: `gemini-stt`=10, `openai-whisper`=20.

**Loader helper:**

```go
// New method on Loader (gateway/internal/upstreams/loader.go).
// Returns tier-1 candidates for a role, ordered by tier_priority ASC (stable).
func (l *Loader) ResolveAllTier1(role string) []UpstreamConfig {
    s := l.snap.Load()
    if s == nil {
        return nil
    }
    var out []UpstreamConfig
    for _, u := range s.ordered {
        if u.Role == role && u.Tier == 1 && u.Enabled {
            out = append(out, u)
        }
    }
    // ordered is already sorted by (role, tier); add a secondary sort by TierPriority.
    sort.SliceStable(out, func(i, j int) bool {
        return out[i].TierPriority < out[j].TierPriority
    })
    return out
}
```

**Dispatcher modification (gateway/internal/proxy/dispatcher.go around line 257-272):**

### Anti-Patterns to Avoid

- **Putting Gemini's API key in `Authorization: Bearer` header.** Gemini rejects `Authorization` headers; it expects `x-goog-api-key`. The director MUST `r.Header.Del("Authorization")` before setting `x-goog-api-key`.
- **Reusing `openai-whisper`'s `BuildOpenAIWhisperDirector` for Gemini.** The wire shapes are incompatible — Gemini is JSON, OpenAI is multipart. A separate director adapter is mandatory.
- **Using tier=2 for the OpenAI safety net.** Violates the binary semantic (tier ∈ {0,1}) that audit dashboards, breaker logs, and shed/schedule middleware assume. Use `tier_priority` instead.
- **Streaming the Gemini response chunk-by-chunk for v1.** Gemini supports `streamGenerateContent` but our `/v1/audio/transcriptions` clients (OpenAI-compatible) DO NOT expect SSE. Stick to non-streaming response (D-B4 scope boundary).
- **Auto-uploading >20MB files via resumable protocol.** Adds 2 round-trips + URL state tracking. Reject with 413/415 for v1 and revisit if observed.

## Common Pitfalls

### Pitfall 1: Multi-tier-1 fallthrough race when both Gemini AND OpenAI breakers flap

**What goes wrong:** Sustained Gemini 429 quota exhaustion drives `gemini-stt` breaker to OPEN → dispatcher falls through to `openai-whisper` → success. But probe loop continues to dial Gemini every 10s. If Gemini intermittently recovers but stays close to quota cliff, breaker oscillates CLOSED↔OPEN, causing 50% of requests to hit Gemini (and 429) and 50% to fall through to OpenAI.

**Why it happens:** Gemini quota error 429 is classified as a failure by `breaker.IsSuccessful`, but it doesn't have the standard "back off N seconds" semantics the breaker assumes — Gemini's quota window is rolling-1-minute, not a sticky cooldown.

**How to avoid:** Set the `gemini-stt` upstream's `circuit_config.cooldown_s` HIGH (e.g. 120s — twice the rolling window) so a single 429 forces a 2-minute fallthrough to OpenAI rather than immediate retry. Document in PLAN 04 + smoke test in Wave 6 by intentionally exhausting Gemini free tier.

**Warning signs:** `gateway_breaker_state{upstream="gemini-stt"} oscillating` in metrics. Audit log shows alternating `upstream=gemini-stt` / `upstream=openai-whisper` on adjacent requests with sub-second latency between them.

### Pitfall 2: Gemini `inline_data` base64 inflation breaks 25 MB request body cap

**What goes wrong:** OpenAI Whisper accepts up to 25 MB audio file. Base64 encoding inflates bytes by ~33% → a 19 MB WAV becomes ~25.3 MB JSON body, hitting our server-level 25 MB cap (mentioned in `openai_whisper_director.go:280` comment).

**Why it happens:** Math: ⌈19 * 4/3⌉ ≈ 25.3 MB encoded.

**How to avoid:** Director rejects audio >18 MB (75% of 25 MB cap, leaving headroom for JSON wrapper + prompt text) via the wrapper handler, returning HTTP 413 with envelope `{"error":{"message":"audio file exceeds 18 MB cap for gemini-stt; use a tier-0 or tier-1 OpenAI Whisper upstream","type":"invalid_request_error"}}`. The breaker stays CLOSED (4xx is non-failure). Dispatcher does NOT auto-fall through to OpenAI because the user-facing request is already error-terminal. **Operator workaround:** retry with a shorter clip.

**Warning signs:** `request_body_size_bytes >= 18_000_000` with `upstream=gemini-stt` in audit log.

### Pitfall 3: `x-goog-api-key` header gets stripped by `clientAuthHeaders` if not whitelisted

**What goes wrong:** `BuildDirector` (parent) strips `Authorization`, `X-API-Key`, `Cookie`, `Proxy-Authorization` from the outbound request (`director.go:32-37`). `x-goog-api-key` is currently NOT in that list, so it survives. BUT if the wrapper accidentally sets `X-API-Key` instead (an easy typo), it gets stripped.

**Why it happens:** Two near-identical header names; Go canonicalizes both to `X-Api-Key` and `X-Goog-Api-Key` respectively — they're different keys. Pass-through tests need to verify the literal header `x-goog-api-key` (case-insensitive at HTTP layer but pinned in test fixtures).

**How to avoid:** Plan 04 director_test MUST assert `r.Header.Get("x-goog-api-key") != ""` AND `r.Header.Get("Authorization") == ""` after the director runs. Add an in-line code comment in director: `// CRITICAL: header name is "x-goog-api-key" NOT "X-API-Key" — the latter is stripped by BuildDirector`.

### Pitfall 4: Speaches HF_HUB_CACHE path mismatch on re-add

**What goes wrong:** Phase 06.8 fix iii (commented in `0f65da5^:supervisord.conf`) required `HF_HUB_CACHE="/weights/whisper"` to point at the pre-shaped HF cache layout extracted by `onstart`. Phase 11.1 dropped this. If revert only restores the supervisord block without verifying the `tar -xzf … -C /weights/whisper` path in `download-weights.sh` matches, speaches boots but hits `get_model_card_data_or_raise()` 500.

**Why it happens:** The HF cache requires a specific directory tree: `/weights/whisper/models--Systran--faster-whisper-large-v3/{refs,snapshots/<hash>/}` including `README.md`. The pre-shaped tarball in MinIO encodes this layout; the extract target must be the cache root.

**How to avoid:** Plan 03 acceptance MUST include a smoke test: start speaches container with the restored config, POST a 1s silent WAV to `:8001/v1/audio/transcriptions`, assert 200 OK + non-empty text. The Phase 06.8 fix-iii comment block from `0f65da5^:supervisord.conf` MUST be restored VERBATIM as documentation in the supervisord file.

**Warning signs:** speaches log: `RuntimeError: Directory 'realtime-console/dist' does not exist` OR `KeyError: ... model card ...`.

### Pitfall 5: Test fixture cascade — 2-role → 3-role assertions across many test files

### Pitfall 6: Gemini API key per-environment scoping (dev vs prod)

**What goes wrong:** Operator generates ONE Gemini API key, uses it in both `vps-ifix-vm` dev `/opt/ai-gateway-dev/.env` AND eventual prod gateway. Shared quota — dev smoke tests consume prod quota budget. Worse: revoke for one revokes for both.

**Why it happens:** Google AI Studio defaults to project-scoped keys; the cap is per-project not per-key.

**How to avoid:** **Operator gate before Wave 0:** create TWO Gemini API keys, one per project (`ifix-ai-gateway-dev`, `ifix-ai-gateway-prod`). Document in plan-04 SEED. Store dev key in `/opt/ai-gateway-dev/.env`; prod key goes in Portainer stack env (when prod stack target chosen — see CLAUDE.md note about decommissioned prod VPS).

### Known Threat Patterns for Go HTTP proxy + Gemini integration

| Pattern | STRIDE | Standard Mitigation |
|---------|--------|---------------------|
| API key leak via logs/audit | Information Disclosure | `json:"-"` tag on AuthBearer struct field; presence-only logging in resolver (see `resolver.go:104` for parity) |
| Sensitive tenant audio routed to external Gemini | Information Disclosure | Existing `data_class='sensitive'` dispatcher path (RES-08) blocks tier-1 for telefonia/cobrancas — applies automatically to gemini-stt because it's a tier-1 row |
| Oversized audio DoS (memory exhaustion on base64) | Denial of Service | 18 MB cap enforced in director before `io.ReadAll`; existing 25 MB request body cap at server level as second wall |
| Header smuggling via `Authorization` cookie injection | Spoofing | `clientAuthHeaders` strip (`director.go:32-37`) already runs via BuildDirector base call |
| Quota exhaustion fail-open | DoS | Multi-tier-1 cascade — Gemini quota 429 → fall through to OpenAI safety net (Pitfall 1 quantifies the breaker tuning) |
| Audio file SSRF via uploaded `file_uri` | Spoofing | Use `inline_data` (base64 inline) — never accept caller-supplied `file_uri`. Documented in Pattern 2 |
