## Plan 08 Fixes (auto-applied during Live UAT)

Three Rule 1 (pre-existing bug) auto-fixes landed during execution:

1. **`fix(11.2-08)` 58590fc — audit_log.upstream cascade distinguishing** (Rule 1).
   `dispatcher.dispatchTo` now stamps the factual upstream name into a
   request-id-keyed sync.Map registry; audit middleware reads/deletes the
   entry post-ServeHTTP. Bypasses `http.TimeoutHandler` request-clone barrier
   that broke prior ctx-propagation attempts. Without this, D-B13 acceptance
   criteria (`audit_log.upstream='gemini-stt'`/`'groq-whisper'`/etc.) is
   machine-unverifiable — every STT request previously wrote upstream='stt'.

2. **`fix(11.2-08)` f641dfb — groq-whisper URL drop trailing `/v1`** (Rule 1).
   `BuildDirector` path-joins upstream env Path with inbound URL — env
   `https://api.groq.com/openai/v1` + `/v1/audio/transcriptions` yielded
   the doubled `/openai/v1/v1/audio/transcriptions` → Groq HTTP 404. Same
   pattern as OpenRouter SEED-004. Fixed default + .env.example + RUNBOOK +
   test fixtures.

3. **`fix(11.2-08)` 9e36ca4 — uat-11.2.sh timestamp-based audit fallback**
   (script-level, not gateway).
   When dispatch target is OpenAI-compat external (Groq), `httputil.ReverseProxy`
   preserves the upstream's `X-Request-ID` response header, clobbering
   the gateway UUID. Script now detects non-UUID request_ids and falls
   back to most-recent-row pick (works because UAT is single-flight).
