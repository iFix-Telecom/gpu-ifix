# Phase 11.2 Plan 06: Gemini STT Director + Cascade Dispatcher + main.go Wireup — Summary

Gateway code path completo da cascade STT 3-upstream tier-1 (gemini-stt → groq-whisper → openai-whisper) implementado e unit-tested. Phase 11.2 Wave 3 fecha o ciclo planning→config→migration→adapter→cascade→wireup; após Plan 06 + Plan 03 deploy, requests `/v1/audio/transcriptions` com pod-OFF cascateiam automaticamente conforme breaker state.

## Tasks Executadas

### Task 1: gemini_stt_director.go (D-B4)

Novo adapter `BuildGeminiSTTDirector` retorna pair `(Director, ModifyResponse)` para `httputil.ReverseProxy`. Translates OpenAI-style multipart audio request → Gemini's `:generateContent` JSON shape e flatten response.

**Director side:**
- `r.Header.Del("Authorization")` + `r.Header.Set("x-goog-api-key", apiKey)` — Pitfall 3 (inline comment: "header name is 'x-goog-api-key' NOT 'X-API-Key' — the latter is stripped by BuildDirector").
- Parse multipart, extract `file` part bytes + MIME via `extractAudioFromMultipart` helper.
- Build `geminiRequest` JSON: `contents[0].parts[]` = `[{text: <prompt>}, {inline_data: {mime_type, data: base64}}]`.
- Rewrite URL path to `/v1beta/models/<model>:generateContent`; model resolvido via `resolver.Resolve("whisper", "gemini-stt")` (env override `UPSTREAM_STT_FALLBACK_1_MODEL` wins, default `gemini-2.5-flash-lite`).
- Content-Type → `application/json`; method → POST.

**ModifyResponse side:**
- Gemini error envelope (`{error: {code, message, status}}`) → translate para 502 + OpenAI envelope `{error: {message, type: "upstream_error", code}}`.
- HTTP 200 + candidates → concatenate `candidates[0].content.parts[*].text` via `strings.Builder` (A5: guard contra chunked transcription) → write `{"text": "<concatenated>"}`.
- Non-200 sem error envelope parseável → pass through unchanged.

**Audio cap:** `geminiMaxInlineBytes = 20 MiB` apenas WARN-logged quando excedido — explicit 413 wrapper deferido a follow-up.

**6 unit tests:**
- `_SetsXGoogApiKeyHeader` — verify header swap.
- `_StripsAuthorizationHeader` — verify Authorization removed.
- `_MultipartToJSON_AudioBytesPreserved` — base64 decode round-trip byte-identical para RIFF/WAVE header fixture.
- `_ResolvesModelViaEnvOverride` — `t.Setenv("UPSTREAM_STT_FALLBACK_1_MODEL", "gemini-2.5-flash")` → URL path contém `gemini-2.5-flash`.
- `_FlattenResponse` — fake upstream replies candidates → client sees `{"text": "transcribed words"}`.
- `_TranslatesGeminiErrorEnvelope` — fake upstream replies 200 + `{error: {code: 429, ...}}` → client sees 502 + `type: "upstream_error"`.

### Task 3: Dispatcher cascade + main.go wireup (D-B5′)

**dispatcher.go:** substituiu single-tier-1 lookup (`Loader.Resolve(role, 1)`) por loop iterando `Loader.ResolveAllTier1(role)` ASC tier_priority. Primeiro candidato com `Breaker.EffectiveState() == StateClosed` ganha. Se lista vazia → 503 "no fallback configured"; se todos OPEN → 503 "All inference upstreams are unavailable".

**Backward compat:** roles llm/embed/tts têm 1 tier-1 row → ResolveAllTier1 retorna slice de tamanho 1, comportamento idêntico ao single-lookup anterior. Zero impacto em tests pre-existentes (validado por suite completa verde).

**main.go:** 2 novos proxies registrados em `sttRoleProxies`:
- `gemini-stt`: `buildGeminiSTTProxy(cfg.UpstreamSTTFallback1URL, cfg.UpstreamSTTFallback1AuthBearer, resolver, log)` — usa BuildGeminiSTTDirector + ModifyResponse + ErrorHandler. Wired ONLY quando loader.Get("gemini-stt") existe E env URL + key populated.
- `groq-whisper`: `buildGroqWhisperProxy(...)` — REUSE de BuildOpenAIWhisperDirector com URL+bearer de `cfg.UpstreamSTTFallback2*`. Wrapped em `WhisperAbortGuard` (parity com openai-whisper).

**3 unit tests cascade** via nova `sttCascadeFixture` (4 backends: local-stt, gemini-stt, groq-whisper, openai-whisper) + `tripBreaker` helper:

## Deviations from Plan

**Plan 06 baseline foi seguido sem auto-fixes Rule 1-3 necessários.** Algumas notas de implementação dentro do escopo planejado:

1. **`extractAudioFromMultipart` helper isolation.** RESEARCH Pattern 2 sugeriu o helper inline; mantido como função separada package-private para legibilidade e para futura reutilização caso novo adapter Anthropic/etc. surja (zero impacto funcional).

2. **`groq-whisper` URL+bearer threaded via env, NOT loader row.** Plan 03 inseriu a row com `url_env_var='UPSTREAM_STT_FALLBACK_2_URL'` e `auth_bearer_env='UPSTREAM_STT_FALLBACK_2_AUTH_BEARER'`. main.go agora reads those via `cfg.UpstreamSTTFallback2URL/AuthBearer` (Plan 02 fields) ao invés de via `loader.Get("groq-whisper")` para evitar dupla resolução. `loader.Get` ainda é consultado como gate (precisa existir no DB para wireup acontecer). Mesma estratégia para gemini-stt. Esse padrão é consistente com openai-whisper que ALREADY threads via `u.AuthBearer` from loader; aqui adicionalmente fallback para env mantém compat se operator não rodou migration.

3. **Auto-mapped `"whisper"` → `geminiDefaultModel` no director.** Quando `resolver.Resolve("whisper", "gemini-stt")` cai no passthrough layer (sem env, sem schema row), retorna `"whisper"` literal. Director detecta esse caso e mapeia para `geminiDefaultModel = "gemini-2.5-flash-lite"`. Plan 03 migration 0029 já inseriu a row `(whisper, gemini-stt, gemini-2.5-flash-lite)`, então em prod isso não dispara — defensivo apenas.

4. **`buildGroqWhisperProxy` constrói `UpstreamConfig` ad-hoc.** Em vez de chamar `loader.Get("groq-whisper")` + reuso, criamos uma `UpstreamConfig{Name, URL, AuthBearer}` minimal apenas com os 3 campos que `buildOpenAIWhisperProxy` lê. Isso isola groq do contrato do schema row caso ele divirja no futuro.

Auth gates: nenhum durante execução (zero env reads em runtime de tests). Operator gate Wave 0 (Gemini + Groq API keys) é responsabilidade de Plan 07/08 deploy gate.
