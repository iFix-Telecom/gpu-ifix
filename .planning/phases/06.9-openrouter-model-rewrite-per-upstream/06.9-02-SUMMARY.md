### Resolve body (resolver.go, full)

```go
// Resolve returns the target for alias on the resolved upstream NAME.
//
// Precedence per D-06 (Phase 06.9):
//  1. Env override via curated upstreamEnvVarMap ‚Äî env wins when non-empty.
//  2. Schema row in model_aliases ‚Äî keyed on (alias, upstream_name).
//  3. Passthrough ‚Äî alias returned unchanged (safety net for new upstreams
//     not yet seeded; pod decides what to do with an unknown model name).
//
// Empty-string env values are treated as unset (do NOT override schema).
//
// Env-override is the documented operator escape hatch ‚Äî see CONTEXT.md
// D-06 for rationale. The CLI `gatewayctl model-alias set` (Plan 04) is
// the multi-instance-consistent override path; env is the per-instance
// path. Both are supported and coequal.
func (r *Resolver) Resolve(alias, upstream string) string {
	// (1) Env-override layer ‚Äî env wins when non-empty.
	if envVar, ok := upstreamEnvVarMap[upstream]; ok {
		if val := os.Getenv(envVar); val != "" {
			return val
		}
	}
	// (2) Schema layer ‚Äî under RLock for concurrent-safe map read.
	r.mu.RLock()
	target, hit := r.aliases[aliasKey{Alias: alias, Upstream: upstream}]
	r.mu.RUnlock()
	if hit {
		return target
	}
	// (3) Passthrough layer.
	return alias
}
```

## Decisions Made

- **Env-override-wins layer lives inside Resolver.Resolve.** Directors (Plan 03) pass the upstream NAME and receive the env-overridden target through the same `resolver.Resolve(alias, upstream)` call ‚Äî no parallel env-read logic in director code, single source of truth.
- **Curated `upstreamEnvVarMap` over reflective discovery.** Three explicit entries match the Wave 0 Gate A operator env contract verbatim. New tier-1 upstreams MUST add a mapping entry alongside their schema row ‚Äî this is the documented contract for tier-1 onboarding. Reflective `UPSTREAM_*_MODEL` discovery would over-include and surprise operators with implicit overrides.
- **Empty env value = unset.** Avoids accidental schema-row collapse when an operator writes `UPSTREAM_LLM_OPENROUTER_MODEL=` (empty assignment) in `.env` rather than `unset`. Mirrors standard 12-factor "absent" semantics.
- **Presence-only observability log.** Every 60s refresh emits one INFO line per active env override, naming the upstream and the env-var name but NEVER the value. Tradeoff: operators can confirm escape-hatch wiring per-tick; logs do not leak operator-overridden slugs (which might encode customer info, internal model names, etc.).
- **Existing `TestResolver_SameAliasDifferentUpstreams` renamed in place rather than added-as-new.** The test already validates the per-upstream-key invariant; only the semantic interpretation of the upstream value changed (role ‚Üí name). Keeping a single test for this invariant under the new semantics is cleaner than carrying both.
- **Handler godoc deprecated, body preserved.** The two `main.go` callers compile and run identically; Plan 03 deletes them when the Directors take over per-upstream rewriting. This is the standard `// Deprecated:` Go idiom (recognized by staticcheck + golangci-lint).
- **No defensive lookup against empty `aliasKey` keys.** When a downstream caller passes empty alias OR empty upstream, the current path naturally falls through to passthrough (returns the alias unchanged). Adding explicit `if alias == "" { return alias }` would be defensive noise ‚Äî the contract already says passthrough on miss.
