## Decisions Made

- **TTL durations in R1 test.** The plan's example used TTL=5s + wait 6s. The R1 test uses TTL=2s + wait 3s — same Redis EX semantics, faster CI. The plan explicitly allowed faster TTLs.
- **R13 reverse sanity in the same test function.** Plan acceptance said "reverse assertion: temporarily wire a `newSelectiveMock(t, acceptedModels=["qwen-NEVER-MATCHES"])` ... inside the same test function". Implemented as Phase B of the same `TestIntegration_HistoricalORFixRegression` test, wiring a separate dispatcher + breaker + classifying proxy to avoid breaker state cross-contamination (the first phase trips local-llm; reusing the same breaker would short-circuit immediately on the second probe before the test could exercise the reject path properly).
- **PROJECT.md surgical insert, not rewrite.** The plan instructed: "Make a SURGICAL update: correct any outdated model slugs ... Do NOT touch other sections." Existing tier-1 mentions in `Active`, `Out of Scope`, `Context`, and `Constraints` sections remain accurate for their contexts (Active: "Qwen 3.5 27B via Novita/Fireworks" — still accurate; Constraints: line 78 "Qwen 3.6 27B Q4_K_M ... b9191 ..." — still accurate). The new `Phase 06.9: Tier-1 Stack Final` section CENTRALIZES the per-upstream model targets table + D-06 env-override-wins documentation without duplicating those touchpoints.
- **Migration 0026 Down-abort row-count assertion is `countAfter == countBefore`.** The migration's Step 1 DELETEs the 3 tier-1 seeded rows, but because Step 2's RAISE EXCEPTION fires inside the SAME transaction, the DELETE rolls back. Row count after the aborted Down therefore equals the pre-Down count (7 in the test: 6 seeded + 1 operator duplicate). This locks the transactional-safety contract: the guard isn't just "tell the operator we aborted", it's "we aborted AND nothing changed".
- **Cross-tier leak test uses LLM env + embed POST (not the reverse).** Test 3 sets the LLM env override and POSTs to `/v1/embeddings` to assert the LLM env does NOT leak into embed. This is the asymmetric leak path that mattered historically; we don't repeat the reverse (embed env + LLM POST) because the test would be a mechanical mirror image — Plan 02's resolver unit tests cover the full matrix.
- **`UPSTREAM_EMBED_OPENAI_MODEL` explicitly set to empty in Test 3.** `t.Setenv("UPSTREAM_EMBED_OPENAI_MODEL", "")` is set alongside the LLM env override to defend against test-runner env pollution: if the CI runner happened to have `UPSTREAM_EMBED_OPENAI_MODEL` set in the environment, the assertion would false-positive in either direction. Explicit empty-set makes the test deterministic.
- **Reverse-sanity rejectMock in R13 uses `tier0Fail` from Phase A.** Phase B reuses `tier0Fail` (already wired in Phase A) instead of creating a fresh failure mock. The local-llm breaker FSM is owned by the SECOND dispatcher's `rejectBs` (fresh breaker.Set), so this reuse is safe: the fresh breaker observes 3× 500 from `tier0Fail` via `driveBreaker` and trips independently of Phase A's breaker.
