<objective>
Wave 1: Refactor the resolver to key on upstream NAME (not role) AND add the D-06 env-override-wins precedence layer. Update the rewrite middleware godoc to reflect deprecation. Wave 0 (migration 0026 + sqlc regen) has already widened the PK and added `UpstreamName` to the generated struct; this plan changes the in-memory cache semantics + the lookup contract + the env-override layer + tests.

Purpose: Make `resolver.Resolve("qwen", "openrouter-chat")` return `"qwen/qwen3.5-27b"` (tier-1 schema slug) while `resolver.Resolve("qwen", "local-llm")` still returns `"qwen"` (tier-0 alias). Honor D-06: when `UPSTREAM_LLM_OPENROUTER_MODEL` is set, env value wins over the schema row at resolver-lookup time. This is the load-bearing behavioral change required for Plan 03 (directors) to do per-upstream model rewriting AND for operators to retain the env-var escape hatch they had before Phase 06.9.

Output: `resolver.go` Refresh loop reads `row.UpstreamName` (the new column from migration 0026). Resolve method consults env first via a curated mapping `upstreamName → envVarName`; if env non-empty, env wins; else schema lookup; else passthrough. New tests `TestResolver_PerUpstreamLookup` + `TestResolver_EnvOverrideWins*` (4 cases) prove the precedence chain end-to-end against in-memory fixture map. `models.Handler` gets a `// Deprecated:` godoc pointer to the per-Director pattern (but the function body is preserved — main.go callers get removed in Plan 03).

Scope: 3 files only. No main.go changes. No director changes. Tests must pass without touching anything outside the resolver package.

**BLOCKER-1 revision (revision iteration 2 — honor D-06 env-override-wins):** Earlier wording said the resolver does NOT consume env vars. CONTEXT.md D-06 actually mandates env-override-wins precedence — env wins over schema row when set. This plan now implements that layer inside `Resolve()`. Operator override paths post-Phase-06.9 are two coequal options (both supported): (a) `gatewayctl model-alias set ...` writes a schema row; (b) `export UPSTREAM_<UPPER_NAME>_MODEL=<target>` + restart container. Plan 04 retains the CLI as the multi-instance-consistent path; this plan retains env as the per-instance escape hatch.

**R9 (revision iteration 2 update):** Cross-plan env-var convergence — `UPSTREAM_LLM_OPENROUTER_MODEL` is the documented runtime fallback override per D-06, NOT deprecated. Plan 01 doc + Plan 04 boot behavior + Plan 05 integration test + Plan 06 UAT all converge on this semantic.
