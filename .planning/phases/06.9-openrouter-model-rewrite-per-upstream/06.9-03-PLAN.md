<objective>
Wave 2: Move model rewriting from request-edge (`models.Handler` in main.go) INTO each per-upstream Director, so the rewrite happens AFTER the dispatcher has selected which upstream gets the request. This is the load-bearing behavioral fix that closes OR-FIX, STT-OAI-FIX, and EMBED-OAI-FIX.

Three tier-1 directors need parallel changes (same shape, three implementations):
1. `BuildOpenRouterDirector` — gains resolver+upstreamName; calls `models.RewriteJSONModel` then existing `injectProviderOrder` + `injectStreamOptionsIncludeUsage`. **R5 honesty:** this is 3 sequential JSON unmarshal+marshal passes, NOT a single pass. We accept this — the code is straightforward and each pass is well-scoped. Refactoring to a single combined helper is deferred.
2. `BuildOpenAIWhisperDirector` — currently passes multipart body through untouched (Phase 02 deferral); gains a NEW multipart-aware `model` form-field rewriter that streams audio bytes via `io.Copy` (never decodes binary as text). **R6:** explicit behaviors for edge cases (missing model field → inject target; duplicate model field → reject with 400; resolver miss → alias passes through unchanged). **WARNING-3 (revision iteration 2): the duplicate-model HTTP 400 abort is WIRED in this phase — no escape hatch, no degraded fallback. Task 2 owns the wire-up surgery.**
3. `BuildOpenAIEmbedDirector` — currently hard-codes `text-embedding-3-small`; refactor to source via resolver (consistency cleanup). `dimensions=1024` stays hard-coded (BGE-M3 parity invariant).

**BLOCKER-1 (revision iteration 2 — D-06 env-override-wins):** Directors call `resolver.Resolve(alias, upstreamName)` directly. The env-override-wins precedence chain (env → schema → passthrough) lives INSIDE `Resolve()` (Plan 02 owns this layer). Directors do NOT contain separate env-read code. When `UPSTREAM_LLM_OPENROUTER_MODEL` is set, the resolver returns that value transparently to the director. Director godocs reference D-06 precedence by linking back to Plan 02 / resolver.go.

**R4 (local-tier coverage NEW per REVIEWS.md MEDIUM):** Local directors (`local-llm` chat, `local-stt` audio, `local-embed` embeddings) are NOT directors per se in this codebase — they're built by `proxy.NewChatProxy` / `proxy.NewEmbeddingsProxy` / `proxy.NewAudioProxy` against `cfg.UpstreamLLMURL` etc. (main.go:552-557). Currently `models.Handler` wraps the chat + embed handlers BEFORE dispatch, providing alias→target rewrite at request edge. Removing those wraps (per this plan's main.go task) means local-tier traffic relies on the Plan 01 backfill: rows `(qwen,llm,qwen,local-llm)` + `(whisper,stt,whisper,local-stt)` + `(bge-m3,embed,bge-m3,local-embed)` provide IDENTITY mappings — but since the local proxies (NewChatProxy etc.) do NOT call the resolver at all, the body is forwarded byte-identical regardless. Decision: local-tier path is PASS-THROUGH (body unchanged, model field forwarded as the client sent it — which is the alias `qwen` / `whisper` / `bge-m3`, exactly what the local pod expects). Per Plan 05a, ADD 2 integration tests asserting local-tier byte-identical forwarding (one for chat, one for embed). No source-code change is required in the local proxies for R4 — the decision is "pass-through, validated by tests."

main.go wiring: delete the 2 `models.Handler` wraps for chat/embed handlers (per Option (a) in 06.9-PATTERNS.md); pass `resolver + upstreamName` into each `Build*Director` constructor at its wire-up site; ALSO add the abort-on-400 wrapper for the Whisper handler chain (WARNING-3).

Purpose: After this plan, gateway request `POST /v1/chat/completions {"model":"qwen"}` with `local-llm` breaker OPEN forwards `{"model":"qwen/qwen3.5-27b", "provider":{"order":["novita"], ...}, ...}` to OpenRouter — proven by 3 new unit tests + Plan 05a integration test. Local-tier paths forward `{"model":"qwen", ...}` byte-identical to the local pod — proven by Plan 05b local-tier byte-identical tests. Duplicate-model multipart requests return HTTP 400 to the client; the request is NEVER forwarded.

**R12 (director error logging):** All 3 directors emit DEBUG/WARN log lines on rewrite errors (e.g., `models.RewriteJSONModel` returns non-nil err) WITHOUT logging the request body. Fields: alias, upstreamName, error class. Sentry breadcrumb on WARN class.

Build succeeds.
